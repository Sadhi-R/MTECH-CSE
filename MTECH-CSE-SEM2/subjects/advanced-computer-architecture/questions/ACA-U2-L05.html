Question

Explain advanced processor performance issues.

DefinitionAdvanced processor performance issues include branch prediction, cache misses, memory wall, ILP limits, and power/thermal constraints.

Visual Example

Introduction

Performance depends on microarchitecture, not clock alone.

Why It Is Needed

Identifying bottlenecks guides compiler and hardware optimization.

Working Principle

Pipeline stalls from branches/misses; memory latency dominates; power caps frequency scaling.

Step-by-Step Explanation

  • Branch prediction: mispredict penalty.
  • Cache hierarchy: L1/L2/L3 miss costs.
  • Memory wall: CPU vs DRAM speed gap.
  • ILP wall: diminishing returns from wider issue.
  • Power/thermal: Dennard scaling ended.

Architecture / Diagram

Pipeline stall on branch miss → flush → refill
Memory: CPU 3GHz vs DRAM ~100ns

Algorithm

Profile → find stalls (branch, cache, memory) → optimize hot path

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Matrix multiply: cache blocking reduces L3 misses 10×; branchy code defeats predictor.

Advantages

  • Understanding guides optimization
  • Explains non-linear scaling
  • Links arch to real perf

Disadvantages

  • Many interacting factors
  • Microarch-specific behavior

Applications

  • Compiler optimization
  • HPC tuning
  • Mobile power management

Exam Tips

List branch + cache + memory wall — three pillars of advanced perf.

Common Mistakes

  • Attributing all slowdown to clock speed only.

Difference Table

ILP wall=instr-level limit | Memory wall=DRAM latency

Time Complexity

Miss penalty dominates effective CPI

Space Complexity

Cache lines, TLB, register file

Interview Questions

  • What ends Dennard scaling?

Memory Tricks

Perf issues = branch + cache + memory wall + power
