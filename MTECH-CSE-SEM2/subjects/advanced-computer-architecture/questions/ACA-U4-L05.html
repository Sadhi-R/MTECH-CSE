Question

Compare multiprocessor and multicomputer systems.

DefinitionMultiprocessor systems share memory (tightly coupled); multicomputer systems have private memory connected by network (loosely coupled).

Visual Example

Introduction

Multiprocessor = SMP/NUMA; multicomputer = cluster of nodes.

Why It Is Needed

Choice affects programming model (OpenMP vs MPI) and scalability.

Working Principle

Multiprocessor: load/store to shared address space. Multicomputer: message passing between address spaces.

Step-by-Step Explanation

  • Multiprocessor: shared memory, cache coherence.
  • Multicomputer: distributed memory, no global address.
  • Programming: OpenMP/pthreads vs MPI.
  • Scalability: MP limited by coherence; MC scales out.
  • Examples: 4-socket server vs HPC cluster.

Architecture / Diagram

Multiprocessor: [CPU CPU CPU]—shared RAM
Multicomputer: [CPU+RAM]—network—[CPU+RAM]

Algorithm

MP: shared load/store | MC: Send/Recv messages

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

OpenMP on 16-core server (MP); MPI on 1000-node cluster (MC).

Advantages

  • MP: easy programming
  • MC: massive scalability
  • MP: low latency sharing

Disadvantages

  • MP: coherence bottleneck
  • MC: explicit communication
  • MP harder to scale beyond sockets

Applications

  • Shared-memory servers
  • HPC clusters
  • Cloud distributed systems

Exam Tips

Side-by-side diagram with programming model — high-value comparison.

Common Mistakes

  • Confusing multicore chip (MP) with cluster node (MC).

Difference Table

Multiprocessor=shared memory | Multicomputer=message passing

Time Complexity

MP: shared mem latency | MC: network latency

Space Complexity

MP: global address | MC: per-node memory

Memory Tricks
