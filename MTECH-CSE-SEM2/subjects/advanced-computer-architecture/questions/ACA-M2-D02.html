Q2: Memory consistency models

High WeightageMid-II Paper100%Unit-3Short
Question

Explain memory consistency models in shared memory systems.

DefinitionMemory consistency model defines what order memory operations appear to execute across multiple processors sharing memory.

Visual Example

Key Points

  • Sequential consistency: all processors see same global order of loads/stores (simplest, slowest).
  • Relaxed models: processor consistency, release consistency, weak ordering — allow reordering for speed.
  • Programs must use memory barriers/fences or synchronization to enforce order when needed.
  • Without consistency rules, CPUs may reorder writes → other cores read stale values.

Real-Time Example

Core 1 writes flag=1 then data=42; Core 2 must see writes in order — consistency model + barriers ensure this.

Exam Keyword Sequential consistency · Relaxed · Barriers · Shared memory

5-Mark Answer (Exam Ready)

Define consistency model. Sequential vs relaxed. Why barriers needed. Shared memory example.

One-Line RevisionConsistency = visible order of memory ops; sequential strictest, relaxed faster with fences.
