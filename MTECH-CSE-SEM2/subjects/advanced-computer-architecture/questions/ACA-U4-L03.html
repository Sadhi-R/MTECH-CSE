Question

Explain synchronization in shared memory systems.

DefinitionSynchronization in shared memory systems coordinates threads via locks, barriers, semaphores, and atomic operations.

Visual Example

Introduction

Hardware provides atomic RMW (compare-and-swap); software builds higher-level sync.

Why It Is Needed

Without sync, race conditions corrupt shared data.

Working Principle

Threads acquire lock before critical section; memory fences ensure visibility; release lock after update.

Step-by-Step Explanation

  • Identify shared variables and critical sections.
  • Use mutex/spinlock for mutual exclusion.
  • Barriers for phase synchronization.
  • Atomics (CAS) for lock-free structures.
  • Avoid deadlock: ordering, timeout.

Architecture / Diagram

Thread A: lock(L)→modify x→unlock(L)
Thread B: waits on L before accessing x

Algorithm

acquire(L) → critical section → release(L); fence ensures store visibility

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Bank account update: two threads deposit; lock prevents lost update race.

Advantages

  • Mutex: simple correctness
  • Barriers: collective sync
  • Atomics: lower overhead when applicable

Disadvantages

  • Lock contention serializes
  • Deadlock if ordering wrong
  • False sharing hurts cache

Applications

  • Multithreaded apps
  • DB transaction locks
  • Parallel numerical codes

Exam Tips

Show race without lock AND fix with mutex — classic exam pattern.

Common Mistakes

  • Using busy-wait spinlock on single-core (wastes cycles).

Difference Table

Mutex=mutual exclusion | Barrier=all threads wait

Time Complexity

Lock acquire O(1) uncontended; contention serializes

Space Complexity

Lock metadata + shared data

Interview Questions

  • Spinlock vs mutex?

Memory Tricks

Sync = locks + barriers + atomics in shared memory
