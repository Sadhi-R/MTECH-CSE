Question

Explain SIMD architecture.

DefinitionSIMD architecture uses one control unit broadcasting instructions to multiple processing elements operating on different data.

Visual Example

Introduction

Examples: GPU warps, AVX units, vector supercomputers (Cray).

Why It Is Needed

Ideal for data-parallel kernels with uniform control flow.

Working Principle

Single instruction decoded once; N parallel lanes execute same op on N data elements.

Step-by-Step Explanation

  • Control unit fetches one instruction.
  • Broadcast to N PEs/lanes.
  • Each PE has own register file slice.
  • Mask/predicate for partial vectors.
  • Memory: unit-stride or gather/scatter.

Architecture / Diagram

[Ctrl Unit]→broadcast→[PE0 PE1 PE2 PE3]
Each PE: ALU + local reg

Algorithm

Fetch instr → broadcast → parallel execute on vector regs → writeback

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

AVX-512: one ADD processes 16 floats; GPU warp: 32 threads same instruction.

Advantages

  • High throughput on arrays
  • Energy efficient for DLP
  • Simple control logic

Disadvantages

  • Divergent branches serialize lanes
  • Setup overhead for short vectors

Applications

  • GPU computing
  • Image/video processing
  • Neural network inference

Exam Tips

Draw control + N PEs diagram; mention lane masking.

Common Mistakes

  • Using SIMD for branch-heavy irregular code.

Difference Table

SIMD=1 control N data paths | MIMD=N controls

Time Complexity

Peak = lanes × FU rate × frequency

Space Complexity

Vector registers per lane

Interview Questions

  • What is lane masking?

Memory Tricks

SIMD = one instruction, many data lanes

5-Mark Answer

Organization diagram. Broadcast. Masking. GPU/AVX example.

10-Mark Answer

SIMD org, control broadcast, masking, memory access, applications.
