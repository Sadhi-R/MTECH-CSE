Memory Hierarchy and Locality

High WeightageQuestion Bank92%Unit-2Long
Question

Explain memory hierarchy and locality principles with AMAT formula.

DefinitionMemory hierarchy arranges storage levels by speed, size, and cost.

Introduction

Caches exploit temporal and spatial locality.

Why It Is Needed

It bridges CPU speed and main-memory latency gap.

Working Principle

Frequent data stays in faster smaller levels for quick access.

Step-by-Step Explanation

  • Draw hierarchy
  • Define locality
  • Compute AMAT

Architecture / Diagram

Pyramid: registers, L1, L2, L3, DRAM, disk.

Algorithm

AMAT = hit time + miss rate * miss penalty.

Flowchart

CPU request -> check L1 -> maybe L2/L3 -> DRAM.

Real-Time Example

Keeping daily grocery items on kitchen counter (cache) instead of store room.

Advantages

  • Large performance gain
  • Cost-effective design

Disadvantages

  • Complex replacement/coherence policies

Applications

  • OS design
  • Compiler optimizations
  • Embedded systems

Exam Tips

Show one numerical AMAT calculation.

Common Mistakes

  • Using hit ratio where miss rate is required.

Difference Table

Hierarchy trades capacity for speed across levels.

Time Complexity

Access time determined by hit/miss statistics.

Space Complexity

Cache metadata overhead per block.

Interview Questions

  • Asked in both written and viva exams.

Memory Tricks

Near data is faster data.
