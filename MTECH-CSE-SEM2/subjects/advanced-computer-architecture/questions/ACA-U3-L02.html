Question

Compare linear and non-linear pipelines.

DefinitionLinear pipeline has one task per stage at a time; non-linear pipeline allows multiple tasks in same stage (feedback/bypass).

Visual Example

Introduction

Pipeline type affects scheduling and throughput calculation.

Why It Is Needed

Non-linear pipelines model real FUs with variable latency and resource conflicts.

Working Principle

Linear: strict flow, no overlap in stage. Non-linear: reservation tables track resource usage per cycle.

Step-by-Step Explanation

  • Define linear pipeline (one task/stage).
  • Define non-linear (multi-cycle stages, feedback).
  • Reservation table for conflict-free scheduling.
  • Compute latency and throughput.
  • Compare scheduling complexity.

Architecture / Diagram

Linear: S1→S2→S3 (one item each)
Non-linear: stage 2 holds 2 cycles, feedback loop

Algorithm

Linear: simple timing | Non-linear: reservation table + collision vector

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Linear: classic 5-stage RISC. Non-linear: FP divide unit reused across iterations.

Advantages

  • Linear: easy analysis
  • Non-linear: models real hardware
  • Reservation table systematic

Disadvantages

  • Non-linear scheduling harder
  • Feedback adds hazard complexity

Applications

  • CPU pipeline design
  • DSP pipelines
  • FP unit scheduling

Exam Tips

Draw reservation table for non-linear — distinguishes strong answers.

Common Mistakes

  • Using linear throughput formula on non-linear pipeline.

Difference Table

Linear=1 task/stage | Non-linear=resource conflicts possible

Time Complexity

Throughput = 1/(cycle time) when full

Space Complexity

Pipeline registers between stages

Interview Questions

  • What is reservation table?

Memory Tricks

Linear=strict flow; Non-linear=feedback/multi-cycle stages

5-Mark Answer

Define both. Reservation table. Throughput/latency. Example.

10-Mark Answer

Both types, reservation table construction, collision vector, examples.
