Question

Explain memory hierarchy and performance impact.

DefinitionMemory hierarchy layers storage from fast/small (registers) to slow/large (disk).

Visual Example

Introduction

Bridges speed-cost gap in computer design.

Why It Is Needed

Fast memory is expensive; hierarchy gives speed of cache with capacity of disk.

Working Principle

Check registers → L1 → L2 → L3 → RAM → disk on each access.

Step-by-Step Explanation

  • Draw pyramid.
  • State size/speed at each level.
  • Explain temporal and spatial locality.
  • Explain cache hit vs miss impact.

Architecture / Diagram

Registers → L1 → L2 → L3 → RAM → Disk (fast to slow)

Algorithm

On access: search hierarchy top-down until hit.

Flowchart

Access → L1 hit? → L2? → ... → RAM → Disk

Real-Time Example

Mobile game: hot data in cache = smooth FPS; misses = stutter.

Advantages

  • Average access near cache speed
  • Cost effective

Disadvantages

  • Cache misses cause stalls
  • Complex coherence in multiprocessors

Applications

  • CPU design
  • Database buffer pools
  • OS paging

Exam Tips

Draw pyramid diagram — frequently asked.

Common Mistakes

  • Saying more RAM always equals faster (locality matters).

Difference Table

Register fastest/smallest · Disk slowest/largest

Time Complexity

Hit: few cycles; Miss: 100+ cycles to RAM

Space Complexity

Hierarchy spans bytes to terabytes

Interview Questions

  • Temporal vs spatial locality?

Memory Tricks

Fast/Small at top, Slow/Big at bottom

5-Mark Answer

Pyramid diagram + locality + performance impact.

10-Mark Answer

Full hierarchy, hit/miss, locality types, example, multiprocessor note.

One-Line RevisionHierarchy = Registers→Cache→RAM→Disk; locality makes it fast.
