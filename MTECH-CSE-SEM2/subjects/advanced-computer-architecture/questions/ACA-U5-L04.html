Question Bank90%Unit-5Long
Question

Compare scalar and vector processing.

DefinitionScalar processing operates on single data elements per instruction; vector processing operates on entire vector registers.

Visual Example

Introduction

Scalar suits control-heavy code; vector suits dense numerical loops.

Why It Is Needed

Modern CPUs intermix scalar and vector instructions.

Working Principle

Scalar: one add per instruction. Vector: one add instruction on N elements in pipeline.

Step-by-Step Explanation

  • Scalar: general purpose, branches OK.
  • Vector: high throughput on arrays.
  • Compare IPC and effective FLOPS.
  • Vector startup overhead vs scalar steady.
  • Auto-vectorization by compiler.

Architecture / Diagram

Scalar: add r1,r2,r3 (1 result)
Vector: vadd vr1,vr2,vr3 (N results)

Algorithm

Profile loop → if dense array, vectorize; else scalar

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Loop of 1M adds: scalar 1M instr; vector ~1M/64 instr with AVX-512.

Advantages

  • Scalar: flexible
  • Vector: N× throughput on arrays
  • Hybrid CPUs best of both

Disadvantages

  • Vector poor on gather/scatter
  • Scalar limited peak FLOPS

Applications

  • Control code vs numeric kernels
  • Compiler vectorization
  • Hand-tuned BLAS

Exam Tips

Quantitative example: 64× speedup potential on vectorizable loop.

Common Mistakes

  • Expecting vector speedup on branchy pointer-chasing code.

Difference Table

Scalar=1 data/instr | Vector=N data/instr

Time Complexity

Vector: up to N× faster on dense loops

Space Complexity

Scalar regs vs vector regs

Interview Questions

  • When won't vectorization help?

Memory Tricks

Scalar=one element; Vector=whole array per instr
