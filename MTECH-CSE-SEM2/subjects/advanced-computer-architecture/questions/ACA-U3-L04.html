Question

Explain arithmetic pipeline.

DefinitionArithmetic pipeline partitions floating-point operations (add, multiply, divide) into stages for overlapped execution.

Visual Example

Introduction

Each stage performs partial computation; multiple operands flow through simultaneously.

Why It Is Needed

Boosts throughput of FP-intensive scientific workloads.

Working Principle

Operands enter stage 1 each cycle; after pipeline fills, one result completes per cycle.

Step-by-Step Explanation

  • Partition FP add/mul into k stages.
  • Insert pipeline registers between stages.
  • Handle hazards: structural, data (RAW).
  • Compute latency k cycles, throughput 1/cycle.
  • Use bypassing for RAW hazards.

Architecture / Diagram

FP Add: Align→Add→Normalize→Round (4 stages)
4 pairs in flight simultaneously

Algorithm

One operand pair enters per cycle after pipeline full

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

4-stage FP adder: 4 operations in flight, 1 result/cycle steady state.

Advantages

  • High FP throughput
  • Amortizes stage delay
  • Standard in vector units

Disadvantages

  • Pipeline fill/drain overhead
  • Data hazards need forwarding

Applications

  • Scientific computing
  • GPU shader cores
  • DSP filters

Exam Tips

State latency vs throughput separately — common exam trap.

Common Mistakes

  • Equating latency with throughput.

Difference Table

Latency=k cycles | Throughput=1 result/cycle (steady)

Time Complexity

Latency k cycles; throughput 1 result/cycle in steady state

Space Complexity

k pipeline registers per operation

Interview Questions

  • Latency vs throughput in FP pipeline?

Memory Tricks

Stage partition. Hazards. Latency/throughput. Diagram.

10-Mark Answer

FP pipeline stages, hazard handling, timing diagram, throughput calc.
