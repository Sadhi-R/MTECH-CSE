Question

Explain Flynn's classification with examples.

DefinitionFlynn's classification categorizes computers by instruction and data stream multiplicity: SISD, SIMD, MISD, MIMD.

Visual Example

Introduction

Proposed by Michael Flynn in 1966 — foundational taxonomy for parallel architecture.

Why It Is Needed

Provides standard vocabulary for describing any parallel system.

Working Principle

Count independent instruction streams (I) and data streams (D); each can be Single (S) or Multiple (M).

Step-by-Step Explanation

  • State four combinations: SISD, SIMD, MISD, MIMD.
  • Define instruction stream and data stream.
  • Give hardware example for each.
  • Note SPMD as practical MIMD subset.
  • Draw 2×2 Flynn table.

Architecture / Diagram

        | Single Data | Multiple Data
Single Instr  | SISD        | SIMD
Multiple Instr| MISD(rare)  | MIMD

Algorithm

Classify system by counting I-streams and D-streams

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

SISD: laptop CPU | SIMD: GPU warp | MISD: fault-tolerant voting | MIMD: cluster

Advantages

  • Universal classification
  • Easy 2×2 table
  • Links to programming models

Disadvantages

  • MISD rarely used alone
  • Hybrid systems blur boundaries

Applications

  • Architecture courses
  • System design
  • HPC classification

Exam Tips

Memorize 2×2 table with one example each — quick 5 marks.

Common Mistakes

  • Saying MISD is common (it's mostly theoretical/pipeline voting).

Difference Table

SISD=1I1D | SIMD=1I many D | MIMD=many I many D

Memory Tricks
