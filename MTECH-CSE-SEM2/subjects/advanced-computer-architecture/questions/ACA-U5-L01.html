Question

Explain vector processing principles.

DefinitionVector processing applies single instructions to long arrays (vectors), exploiting data-level parallelism via pipelined functional units.

Visual Example

Introduction

Vector registers hold N elements; vector instructions operate on all elements in pipeline fashion.

Why It Is Needed

Scientific codes with inner loops over arrays achieve near-peak FLOPS.

Working Principle

Load vector → execute vector op in pipelined FU → store result; strip-mining handles long arrays.

Step-by-Step Explanation

  • Define vector length (VL) and vector registers.
  • Vector load/store (unit stride).
  • Pipelined vector add/mul FUs.
  • Strip-mine loops longer than VL.
  • Chaining: result feeds next op without store.

Architecture / Diagram

VR1=[a1..a64] + VR2=[b1..b64] → VR3 (pipelined add)

Algorithm

for i=0; i

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

SAXPY y=a*x+y on 1M elements: vector unit processes 64/cycle.

Advantages

  • High sustained FLOPS
  • Amortizes instruction fetch
  • Chaining reduces memory traffic

Disadvantages

  • Poor on irregular/indirect access
  • Vector startup overhead

Applications

  • Supercomputers
  • BLAS Level 1/2
  • GPU SIMT kernels

Exam Tips

Mention strip-mining and chaining — exam differentiators.

Common Mistakes

  • Ignoring vector startup latency for short loops.

Difference Table

Vector=DLP on arrays | Scalar=one element/op

Time Complexity

O(n/pipeline_rate) sustained

Space Complexity

Vector registers + mask
