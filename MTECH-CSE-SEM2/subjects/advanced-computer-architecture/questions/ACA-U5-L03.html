Question

Explain CM-5 architecture.

DefinitionCM-5 (Connection Machine 5) is a MIMD supercomputer with SPMD programming, fat-tree network, and optional vector units.

Visual Example

Introduction

Designed by Thinking Machines; combined data-parallel processing with message passing.

Why It Is Needed

Landmark architecture bridging SIMD thinking and scalable MIMD clusters.

Working Principle

Thousands of SPARC-based nodes on fat-tree; global sync via network; host front-end for I/O.

Step-by-Step Explanation

  • Processing nodes with local memory.
  • Fat-tree interconnect for low diameter.
  • Virtual channel routing.
  • Data parallel ops via CM Fortran.
  • Host + I/O system.

Architecture / Diagram

Host → Fat-tree → [Node Node Node ... Node]
Each node: SPARC + mem + router

Algorithm

SPMD program on all nodes; CMMD message passing for communication

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

1024-node CM-5 running parallel FFT with fat-tree all-to-all.

Advantages

  • Scalable fat-tree network
  • SPMD + data parallel hybrid
  • Proven HPC architecture

Disadvantages

  • Expensive custom hardware
  • Discontinued product line

Applications

  • Historical HPC
  • Parallel algorithm research
  • Fat-tree network design

Exam Tips

Draw fat-tree connecting processing nodes — key CM-5 feature.

Common Mistakes

  • Calling CM-5 pure SIMD (it's MIMD with SPMD).

Difference Table

CM-5=MIMD nodes + fat-tree + SPMD

Time Complexity

O(log N) network diameter typical

Space Complexity

Per-node local memory

  • What is fat-tree interconnect?

Memory Tricks
