Question

Explain interconnect architectures.

DefinitionInterconnect architectures link processors, memory, and I/O — topology determines bandwidth, latency, and scalability.

Visual Example

Introduction

Types: bus, crossbar, multistage, mesh, torus, hypercube, and fat-tree.

Why It Is Needed

Interconnect is often the bottleneck in parallel performance.

Working Principle

Processors exchange messages or access shared memory through switches, buses, or direct links.

Step-by-Step Explanation

  • Classify: static vs dynamic, shared vs distributed.
  • Describe bus, crossbar, omega, mesh, torus.
  • State diameter and bisection bandwidth.
  • Draw small mesh/torus/hypercube.
  • Relate to NUMA and clusters.

Architecture / Diagram

Mesh 2D: each node 4 links
Torus: wrap-around edges
Fat-tree: tree with fatter links near root

Algorithm

Pick topology by node count, cost, and communication pattern

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

16-node cluster on 4×4 mesh; supercomputer on 3D torus (IBM Blue Gene style).

Advantages

  • Crossbar: low latency, high cost
  • Mesh/torus: scalable
  • Fat-tree: good for switches

Disadvantages

  • Bus: contention bottleneck
  • High-diameter mesh: long hops

Applications

  • Multiprocessors
  • Clusters
  • On-chip NoC
  • Supercomputers

Exam Tips

Draw 4-node mesh AND state bisection bandwidth — strong 10-mark answer.

Common Mistakes

  • Ignoring bisection bandwidth when comparing topologies.

Difference Table

Bus=shared wire | Crossbar=full switch | Mesh=grid links

Time Complexity

Latency O(diameter); bandwidth O(bisection)

Space Complexity

Switch buffers + routing tables

Interview Questions

  • What is bisection bandwidth?

Memory Tricks

Interconnect = how PEs talk — bus, mesh, torus, fat-tree

5-Mark Answer

Topology types. Diagrams. Metrics (diameter, bisection). Examples.

10-Mark Answer

Full survey of interconnects with diagrams, metrics, cost-scalability trade-offs.

One-Line RevisionInterconnect topologies (bus, crossbar, mesh, torus) define parallel system communication.
