PRAM Models

High WeightageQuestion Bank93%Unit-1Long
Question

Explain PRAM variants EREW, CREW, and CRCW with read/write conflict handling.

DefinitionPRAM is an ideal shared-memory parallel model with synchronized processors.

Introduction

Variants differ by simultaneous memory read/write permissions.

Why It Is Needed

These models simplify design and analysis of parallel algorithms.

Working Principle

Processors execute lock-step operations on shared memory.

Step-by-Step Explanation

  • Define PRAM assumptions
  • Explain EREW/CREW/CRCW
  • Give conflict examples

Architecture / Diagram

Processor-memory block diagram with access rules.

Algorithm

Model-level pseudo algorithm under synchronous steps.

Flowchart

Step t: all processors execute -> memory access rules apply.

Real-Time Example

Many students reading same online notes (CREW) but one can edit row.

Advantages

  • Clear theoretical baseline
  • Enables complexity proofs

Disadvantages

  • Unrealistic memory contention assumptions

Applications

  • Parallel algorithm theory
  • Complexity classes NC

Exam Tips

Mention concurrent write resolution policies in CRCW.

Common Mistakes

  • Ignoring synchronization barrier between steps.

Difference Table

EREW is strictest, CRCW most permissive.

Time Complexity

Often O(T(n)) parallel time with p processors in model.

Space Complexity

Shared memory size depends on algorithm.

Interview Questions

  • Interviewers ask for EREW vs CREW difference.

Memory Tricks

E=exclusive, C=concurrent, RW read/write permissions.

5-Mark Answer

State assumptions and three variants.

10-Mark Answer

Show one operation under each access rule.
