Question

Explain memory consistency models.

DefinitionMemory consistency models define what values loads may return when multiple processors access shared memory concurrently.

Visual Example

Introduction

Models: sequential consistency (SC), relaxed (TSO, PSO), release consistency, and C++11 memory order.

Why It Is Needed

Correct model ensures parallel programs behave as intended without excess fences.

Working Principle

Hardware may reorder loads/stores; consistency model specifies allowed orderings and fence requirements.

Step-by-Step Explanation

  • Define sequential consistency (program order + interleaving).
  • Explain relaxed: TSO, weak ordering.
  • Release/acquire semantics for sync.
  • Fence instructions enforce ordering.
  • Compare performance vs ease of programming.

Architecture / Diagram

SC: all loads see same global order
Relaxed: stores may buffer (TSO)

Algorithm

Program with locks/fences → model guarantees visibility

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Two cores: without fence, store may not be visible immediately (TSO); fence ensures publish.

Advantages

  • Relaxed: higher performance
  • SC: easiest to reason
  • Fences target hot paths only

Disadvantages

  • Relaxed models hard to program
  • Subtle bugs without fences

Applications

  • Multicore CPUs
  • GPU memory model
  • Lock-free data structures

Exam Tips

Define SC first, then why relaxed models exist — standard exam flow.

Common Mistakes

  • Assuming all systems are sequentially consistent.

Difference Table

SC=strictest | TSO=store buffer | Weak=most relaxed

Time Complexity

Fences add stall cycles

Space Complexity

Store buffers, invalidate queues

Interview Questions

  • Difference SC vs TSO?

Memory Tricks

Consistency = what order loads/stores visible
