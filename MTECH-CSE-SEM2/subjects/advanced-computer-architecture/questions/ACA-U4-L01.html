Question

Explain cache coherence protocols.

DefinitionCache coherence ensures all CPU caches show the same value for shared data.

Visual Example

Introduction

Multiprocessors share memory — each core has private cache.

Why It Is Needed

Without coherence, cores see different values → program errors.

Working Principle

Snooping protocols track line states: Modified, Shared, Invalid (MSI/MESI).

Step-by-Step Explanation

  • Describe problem: CPU1 writes, CPU2 stale.
  • Explain MSI states.
  • Write invalidation on write.
  • Give snooping bus example.

Architecture / Diagram

CPU1 cache M | CPU2 cache I after write

Algorithm

On write: invalidate other copies. On read miss: fetch from owner or memory.

Flowchart

Read hit/miss → Write → Invalidate others → Update state

Real-Time Example

Two cores updating shared counter — coherence keeps count correct.

Advantages

  • Correct shared-memory programs
  • Hardware-transparent to programmer

Disadvantages

  • Bus traffic
  • Complexity in large systems

Applications

  • Multicore desktops
  • SMP servers
  • NUMA machines

Exam Tips

Know MSI/MESI state names.

Common Mistakes

  • Confusing coherence with consistency (ordering).

Difference Table

Coherence=same value | Consistency=operation ordering

Time Complexity

Protocol-dependent

Space Complexity

Directory or snoop metadata
