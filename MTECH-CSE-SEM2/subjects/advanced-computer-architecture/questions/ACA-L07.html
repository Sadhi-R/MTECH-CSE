Cache Coherence Protocols MSI/MESI

High WeightageMid-II Q295%Unit-3Long
Question

Explain cache coherence problem and MSI/MESI protocols.

DefinitionCache coherence ensures all cores observe consistent memory values.

Introduction

Private caches can hold stale shared data without coherence.

Why It Is Needed

Critical for multicore correctness.

Working Principle

Protocol tracks line states and bus transactions to maintain consistency.

Step-by-Step Explanation

  • Define coherence issue
  • Explain MSI states
  • Add MESI optimization
  • Give transition example

Architecture / Diagram

State diagram with Modified, Shared, Invalid, Exclusive.

Algorithm

State transition algorithm triggered by read/write snoop events.

Flowchart

Core read/write -> bus action -> state transition -> data served.

Real-Time Example

Two bank clerks editing same account balance must sync updates.

Advantages

  • Correct shared-memory behavior
  • MESI reduces unnecessary write-backs

Disadvantages

  • Bus traffic overhead
  • State machine complexity

Applications

  • Multicore CPUs
  • Server processors

Exam Tips

Mention write-invalidate approach explicitly.

Common Mistakes

  • Confusing coherence with memory consistency model.

Difference Table

Time Complexity

Space Complexity
