Explain scalable performance in parallel systems.

Question Bank90%Unit-2Long
Question

Explain scalable performance in parallel systems.

DefinitionScalable performance means parallel speedup grows reasonably as processor count increases — governed by Amdahl's and Gustafson's laws.

Visual Example

Introduction

Scalability measures how efficiency changes with system size.

Why It Is Needed

HPC and cloud systems must scale to justify adding more nodes.

Working Principle

Speedup S(p)=T1/Tp; efficiency E=S/p; limited by serial fraction and communication.

Step-by-Step Explanation

  • Define speedup, efficiency, scalability.
  • Amdahl: S≤1/(s+(1-s)/p).
  • Gustafson: scaled problem size grows with p.
  • Identify serial bottleneck and communication.
  • Plot speedup vs p; note saturation.

Architecture / Diagram

Speedup curve: rises then flattens due to serial fraction s

Algorithm

Measure T1 and Tp → compute S and E → identify limiting factor

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

95% parallel code on 64 cores: Amdahl caps speedup ~20× if s=0.05.

Advantages

  • Quantifies parallel benefit
  • Guides optimization targets
  • Compares systems fairly

Disadvantages

  • Strong scaling hits memory/comm limits
  • Efficiency drops at large p

Applications

  • Supercomputer procurement
  • Cloud auto-scaling
  • Algorithm tuning

Exam Tips

Write Amdahl formula AND interpret serial fraction — essential.

Common Mistakes

  • Claiming linear speedup always (ignoring serial parts).

Difference Table

Amdahl=fixed problem | Gustafson=growing problem

Time Complexity

T(p) includes compute + communication

Space Complexity

Replicated or partitioned data

Interview Questions

  • Strong vs weak scaling?

Memory Tricks
