Flynn's Taxonomy

High WeightageMid-I Q195%Unit-1Long
Question

Explain Flynn's taxonomy and classify SISD, SIMD, MISD, MIMD with examples.

DefinitionFlynn classifies computers by number of concurrent instruction and data streams.

Introduction

It is a foundational model to understand parallel architectures.

Why It Is Needed

Helps map workloads to right hardware style.

Working Principle

Check instruction stream count and data stream count.

Step-by-Step Explanation

  • Define four classes
  • Give hardware examples
  • Compare use cases

Architecture / Diagram

2x2 matrix with instruction streams vs data streams.

Algorithm

Classification procedure based on stream multiplicity.

Flowchart

Input architecture -> identify streams -> classify -> justify.

Real-Time Example

SIMD like restaurant kitchen where one chef command serves many plates.

Advantages

  • Simple conceptual model
  • Useful for exam comparisons

Disadvantages

  • MISD rare in practical systems
  • Oversimplifies modern hybrids

Applications

  • CPU design
  • HPC selection
  • Embedded systems

Exam Tips

Write one practical example for each class.

Common Mistakes

  • Mixing SIMD and MIMD definitions.

Difference Table

SIMD: one instruction many data; MIMD: many instructions many data.

Time Complexity

Classification only, no runtime complexity.

Space Complexity

Not applicable.

Interview Questions

  • Asked in interviews for architecture basics.

Memory Tricks

Flynn = count instruction/data streams.

5-Mark Answer

Draw taxonomy table and define each class.

10-Mark Answer

Include examples from CPU, GPU, and fault-tolerant setups.
