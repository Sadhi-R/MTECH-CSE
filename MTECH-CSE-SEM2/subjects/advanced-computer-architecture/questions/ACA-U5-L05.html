Question

Explain compound vector processing.

DefinitionCompound vector processing chains multiple vector operations without intermediate memory stores — fusing ops in pipeline.

Visual Example

Introduction

Also called vector chaining or fused multiply-add (FMA) pipelines.

Why It Is Needed

Reduces memory bandwidth pressure and increases effective FLOPS.

Working Principle

Result of one vector op directly feeds next op's operand register in same vector pipeline.

Step-by-Step Explanation

  • Execute vector op 1 in pipeline.
  • Result register feeds op 2 directly (chaining).
  • No memory round-trip between ops.
  • FMA: a*b+c in one chained operation.
  • Compiler/hardware detects chainable sequences.

Architecture / Diagram

LOAD → VMUL → VADD → STORE
Chained: VMUL result → VADD input directly

Algorithm

Detect independent chain: r3=r1*r2; r4=r3+r5 → fuse pipeline

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

DAXPY-like: temp= a*x; y= temp+y chained without storing temp.

Advantages

  • Higher effective throughput
  • Lower memory bandwidth
  • FMA improves accuracy

Disadvantages

  • Limited by pipeline length
  • Not all sequences chainable

Applications

  • BLAS optimization
  • GPU FMA units
  • Scientific kernels

Exam Tips

Draw chained pipeline vs store-reload — shows understanding.

Common Mistakes

  • Assuming all vector ops automatically chain.

Difference Table

Chained=no intermediate store | Unchained=mem round trip

Time Complexity

Chaining hides latency between dependent vector ops

Space Complexity

Vector register file holds intermediates

Interview Questions

  • What is FMA in vector context?

Memory Tricks

Compound vector = chain ops without memory store

Chaining concept. FMA. Pipeline diagram. Bandwidth savings.
