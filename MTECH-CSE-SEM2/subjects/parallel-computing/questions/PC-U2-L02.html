Question

Explain decomposition strategies.

DefinitionDecomposition strategies split computation/data into tasks: domain (geometric), functional (pipeline), recursive (divide-conquer), and block-cyclic.

Visual Example

Introduction

Choice affects locality, load balance, and communication volume.

Why It Is Needed

Correct decomposition is the most critical parallel design decision.

Working Principle

Assign disjoint or overlapping data subsets to processors; balance grain size vs overhead.

Step-by-Step Explanation

  • Domain: split grid/matrix by rows/cols/blocks.
  • Functional: different stages on different PEs.
  • Recursive: halve problem (parallel merge sort).
  • Block-cyclic: balance irregular work.
  • Evaluate comm volume and load balance.

Architecture / Diagram

8×8 matrix row-block: P0 rows 0-1, P1 rows 2-3, ...

Algorithm

Analyze data access pattern → pick decomposition minimizing remote access

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

2D block decomposition for matrix multiply minimizes communication vs 1D row split.

Advantages

  • Domain: good locality
  • Functional: pipeline throughput
  • Block-cyclic: load balance

Disadvantages

  • Functional: pipeline bubbles
  • Fine decomposition: high overhead
  • Bad split: all-to-all comm

Applications

  • FDM simulations
  • Matrix operations
  • Image tiling

Exam Tips

Compare 1D row vs 2D block decomposition for matmul.

Common Mistakes

  • 1D decomposition when 2D would halve communication.

Difference Table
