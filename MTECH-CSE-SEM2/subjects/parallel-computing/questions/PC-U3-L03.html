Question

Explain MPI architecture.

DefinitionMPI architecture layers user API over MPI library, network drivers, and hardware interconnect.

Visual Example

Introduction

Includes process manager, communication engine, and optional collective algorithms module.

Why It Is Needed

Understanding architecture explains performance and portability.

Working Principle

Application calls MPI API → library packs messages → NIC transfers → remote MPI unpacks → user buffer.

Step-by-Step Explanation

  • MPI process manager launches p processes.
  • Rank-to-node mapping.
  • Communication device (TCP, InfiniBand).
  • Collective module (tree, ring algorithms).
  • PMPI profiling interface.

Architecture / Diagram

App → MPI lib → Socket/IB driver → Network → Remote MPI → App

Algorithm

MPI_Init spawns/finds processes; comm layer routes messages

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

OpenMPI on Linux cluster using InfiniBand for low-latency MPI_Send.

Advantages

  • Layered portability
  • Optimized collectives per network
  • Standard ABI across vendors

Disadvantages

  • Vendor differences in tuning
  • Process startup overhead

Applications

  • HPC clusters
  • Cloud MPI (Intel MPI, OpenMPI)
  • Containerized parallel jobs

Exam Tips

Draw layered architecture diagram — often asked.

Common Mistakes

  • Assuming MPI bypasses OS network stack entirely (usually goes through driver).

Difference Table

MPI=library layer | Raw sockets=lower level

Time Complexity

Latency α + bandwidth β per message

Space Complexity

MPI buffers + NIC queues

Interview Questions

  • Role of process manager in MPI?

Memory Tricks

MPI arch = API → lib → driver → network

5-Mark Answer

Layers. Process manager. Network. Collective module.

10-Mark Answer

MPI architecture: API, library, process manager, network layer, collective algorithms.
