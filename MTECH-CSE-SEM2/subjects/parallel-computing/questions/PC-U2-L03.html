Question

Explain load balancing techniques.

DefinitionLoad balancing distributes work evenly across processors to minimize idle time.

Visual Example

Introduction

Static: predetermined assignment. Dynamic: work queue, work-stealing, or task migration.

Why It Is Needed

Critical when workload is irregular or input-dependent.

Working Principle

Measure load per PE; redistribute tasks from overloaded to underloaded processors.

Step-by-Step Explanation

  • Static: block/cyclic partition upfront.
  • Dynamic: central/global task queue.
  • Work-stealing: idle PE takes from busy neighbor.
  • Semi-static: periodic redistribution.
  • Metrics: load imbalance factor, idle time.

Architecture / Diagram

Busy PEs with long queues; idle PE steals half the tasks

Algorithm

if idle: steal from busiest queue; else process local tasks

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Parallel tree search: dynamic queue assigns unexplored nodes to idle workers.

Advantages

  • Static: zero runtime overhead
  • Dynamic: handles irregularity
  • Work-stealing: decentralized

Disadvantages

  • Dynamic: queue contention
  • Migration cost
  • Static fails on irregular input

Applications

  • N-body simulation
  • Branch-and-bound
  • Web crawling
  • OpenMP dynamic schedule

Exam Tips

Define static vs dynamic with when-to-use — exam staple.

Common Mistakes

  • Static balancing for highly irregular workloads.

Difference Table

Static=fixed partition | Dynamic=runtime reassignment

Time Complexity

Goal: minimize max(PE load) - min(PE load)

Space Complexity

Task queues per PE

Interview Questions

  • Work-stealing vs central queue?

Memory Tricks

Load balance = even work across PEs

5-Mark Answer

Static/dynamic/stealing. When each. Example. Metrics.

10-Mark Answer

Load balancing: static (regular), dynamic/work-stealing (irregular workloads).

One-Line RevisionLoad balancing evens work via static partition or dynamic/work-stealing assignment.
