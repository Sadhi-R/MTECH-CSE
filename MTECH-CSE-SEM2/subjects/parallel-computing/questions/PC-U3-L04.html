Question Bank90%Unit-3Long
Question

Explain thread creation and lifecycle.

DefinitionThread creation and lifecycle in POSIX/OpenMP: create (pthread_create/parallel), run, join/wait, and terminate.

Visual Example

Introduction

Threads share address space; lifecycle managed by OS scheduler and runtime.

Why It Is Needed

Correct lifecycle prevents leaks and race conditions.

Working Principle

Main creates threads → threads execute start routine → join waits for completion → resources freed.

Step-by-Step Explanation

  • Create: pthread_create or #pragma omp parallel.
  • Runnable: OS schedules on cores.
  • Running: executes user function.
  • Blocked: waiting on lock/condition.
  • Terminated: join reclaims resources.

Architecture / Diagram

Main → create T1,T2 → run parallel → join → exit

Algorithm

pthread_create(&tid, NULL, func, arg); ... pthread_join(tid, NULL);

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

OpenMP: #pragma omp parallel for creates team, implicit barrier at end.

Advantages

  • Lightweight vs processes
  • Shared memory easy
  • OpenMP simplifies creation

Disadvantages

  • Race if unsynchronized
  • Thread leak if no join
  • Oversubscription hurts perf

Applications

  • Multicore apps
  • Server thread pools
  • Parallel loops

Exam Tips

State diagram: New→Runnable→Running→Blocked→Terminated.

Common Mistakes

  • Not joining detached threads causing resource leak.

Difference Table

Thread=shared memory | Process=private memory

Time Complexity

Create/join O(1); oversubscription adds context switch cost

Space Complexity

Shared heap + per-thread stack

Interview Questions

  • join vs detach?

Memory Tricks

Thread life = create → run → join → terminate
