Question

Explain Dijkstra algorithm with example.

DefinitionDijkstra's algorithm finds single-source shortest paths in graphs with non-negative edge weights using a greedy Min-Heap.

Visual Example

Introduction

Standard algorithm for weighted shortest path (GPS, routing).

Why It Is Needed

BFS fails when edges have different costs.

Working Principle

Always settle the unvisited node with minimum tentative distance; relax its neighbors.

Step-by-Step Explanation

  • dist[source]=0, others=∞.
  • Insert (0,source) in min-heap.
  • Extract min (d,u); skip if stale.
  • For each edge (u,v,w): if dist[u]+w < dist[v], update and insert.
  • Repeat until heap empty.

Algorithm

Priority queue + relaxation: dist[v]=min(dist[v], dist[u]+w)

Flowchart

Init → extract min → relax neighbors → repeat

Real-Time Example

Food delivery: find fastest route when roads have different travel times.

Advantages

  • Efficient with heap
  • Greedy and intuitive
  • Works on sparse graphs

Disadvantages

  • No negative weights
  • Heap operations add log factor

Applications

  • Google Maps routing
  • Network routing
  • Game AI pathfinding

Exam Tips

State heap as data structure. Fill-in: single-source shortest path.

Common Mistakes

  • Using with negative edges
  • not marking stale heap entries.

Difference Table

Dijkstra=non-negative+heap | Bellman-Ford=handles negatives, slower

Time Complexity

O((V+E) log V) with binary heap

Space Complexity

O(V)

Interview Questions

  • Why greedy works? When Bellman-Ford instead?

Memory Tricks

Dijkstra = Distance + Heap, No Negatives
