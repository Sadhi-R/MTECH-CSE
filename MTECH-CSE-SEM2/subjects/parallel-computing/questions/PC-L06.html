Parallel Sorting Approaches

Question Bank88%Unit-3Long
Question

Explain odd-even transposition sort and bitonic sort in parallel setting.

DefinitionParallel sorting divides data among processors and performs coordinated compare-exchange phases.

Introduction

Odd-even and bitonic are classic network-style parallel sorts.

Why It Is Needed

They illustrate communication/computation interplay.

Working Principle

Processors exchange border elements and locally sort in rounds.

Step-by-Step Explanation

  • Describe odd-even phases
  • Describe bitonic merge
  • Compare complexity

Architecture / Diagram

Sorting network diagram with compare-exchange wires.

Algorithm

Phase-based compare-swap schedule across processors.

Flowchart

Local sort -> exchange -> merge -> repeat until global order.

Real-Time Example

Multiple counters sorting token numbers in coordinated passes.

Advantages

  • Regular structure
  • Easy hardware mapping

Disadvantages

  • Communication-heavy for large distributed datasets

Applications

  • GPU sorting
  • Network processors
  • Parallel libraries

Exam Tips

Add phase count and communication per phase.

Common Mistakes

  • Skipping initial local sort before exchanges.

Difference Table

Bitonic works best on power-of-two sizes with structured network.

Time Complexity

Varies by network; often O((n/p)log(n/p)+communication).

Space Complexity

Needs local arrays plus exchange buffers.

Interview Questions

  • Occasional interview topic in HPC roles.

Memory Tricks

Sort locally, then repeatedly exchange and merge.

5-Mark Answer

Explain both algorithms with phase diagrams.

10-Mark Answer

Compare communication cost and scalability.
