Scalability Metrics in Parallel Programs

High WeightageMid-II Q392%Unit-5Long
Question

Explain efficiency, scalability, and cost-optimality in parallel algorithms.

DefinitionEfficiency is speedup per processor; cost is p*Tp; cost-optimal means cost approximates serial time.

Introduction

These metrics assess whether added processors are worth it.

Why It Is Needed

Essential for selecting practical processor count.

Working Principle

Compute speedup, efficiency, and overhead for increasing p.

Step-by-Step Explanation

  • Define metrics
  • Show relation with overhead
  • Interpret scaling plot

Architecture / Diagram

Efficiency curve dropping as processors increase.

Algorithm

E = S/p, Cost = p*Tp, Overhead = p*Tp - T1.

Flowchart

Run benchmark at different p -> compute metrics -> choose p.

Real-Time Example

Adding too many chefs in small kitchen causes coordination overhead.

Advantages

  • Prevents over-provisioning
  • Quantifies parallel quality

Disadvantages

  • Needs careful benchmark methodology

Applications

  • Cloud budgeting
  • HPC job planning
  • Parallel library tuning

Exam Tips

State fixed-size vs scaled-size experiment context.

Common Mistakes

  • Judging scalability only by speedup without efficiency.

Difference Table

Strong scaling keeps problem fixed; weak scaling grows problem with p.

Time Complexity

Formula computation O(1) per data point.

Space Complexity

Minimal extra space.

Interview Questions

  • Frequently asked in performance roles.

Memory Tricks

More processors help only if efficiency stays useful.

5-Mark Answer

Define and relate all three metrics.

10-Mark Answer

Interpret sample data and draw conclusion on optimum p.
