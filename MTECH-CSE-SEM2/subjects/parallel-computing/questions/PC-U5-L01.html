Question

Explain Prim algorithm with example.

DefinitionPrim's algorithm builds MST by growing tree from start vertex, always adding minimum-weight edge to fringe.

Visual Example

Introduction

Greedy: at each step pick cheapest edge connecting tree to non-tree vertex.

Why It Is Needed

Used for network design — minimum cost connecting all nodes.

Working Principle

Maintain set in-tree; priority queue of fringe edges; extract min, add vertex.

Step-by-Step Explanation

  • Start from arbitrary vertex s.
  • Add all edges from s to PQ.
  • Extract min edge (u,v) where v not in tree.
  • Add v and its outgoing fringe edges.
  • Repeat until V-1 edges selected.

Architecture / Diagram

Step: {A}→add edge A-B(2)→{A,B}→add B-C(3)...

Algorithm

Prim(G,s): while |T|

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

4 cities: A-B(1), B-C(2), C-D(3), A-D(10) → MST cost 6.

Advantages

  • Greedy optimal for MST
  • O(E log V) with binary heap
  • Dense graphs variant O(V²)

Disadvantages

  • Slow on very sparse graphs without heap
  • Not for shortest path

Applications

  • Network cabling
  • Cluster wiring
  • Approximation schemes

Exam Tips

Trace small graph step-by-step with fringe PQ — standard 10-mark.

Common Mistakes

  • Adding edge that creates cycle.

Difference Table

Prim=grow tree | Kruskal=sort all edges

Time Complexity

O(E log V) with heap

Space Complexity

O(V) in-tree set + PQ
