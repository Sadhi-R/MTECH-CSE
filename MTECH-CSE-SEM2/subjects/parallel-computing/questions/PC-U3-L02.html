Question

Explain PThreads synchronization mechanisms.

DefinitionPThreads provides threads sharing one address space. Sync via mutex, semaphore, condition variables.

Visual Example

Introduction

Shared-memory parallelism on multicore.

Why It Is Needed

Threads lighter than processes; fast shared data access.

Working Principle

pthread_create starts thread; mutex protects critical section; join waits for finish.

Step-by-Step Explanation

  • Create thread.
  • Lock mutex before shared update.
  • Unlock after.
  • pthread_join to wait.

Architecture / Diagram

Main thread + worker threads → shared memory region (protected by mutex)

Algorithm

pthread_create / mutex_lock / unlock / join

Flowchart

Create threads → parallel work → lock shared section → unlock → join

Real-Time Example

Multiple cashiers updating one shared inventory count safely with mutex.

Advantages

  • Fast shared memory
  • Lightweight vs processes

Disadvantages

  • Race conditions without sync
  • Harder to debug

Applications

  • Multicore apps
  • Web servers
  • Game engines

Exam Tips

PThreads=shared address; MPI=separate addresses.

Common Mistakes

  • Forgetting to unlock mutex (deadlock).

Difference Table

Mutex=binary lock | Semaphore=counting lock

Time Complexity

Thread ops microseconds

Space Complexity

Shared heap + per-thread stack

Interview Questions

  • Mutex vs semaphore?

Memory Tricks

MPI=Messages, PThreads=Shared memory

5-Mark Answer

Thread creation, mutex, critical section, join.

10-Mark Answer

Architecture, code sketch, mutex vs semaphore, race example, deadlock note.

One-Line RevisionPThreads=shared memory threads; Mutex=one-at-a-time critical section.
