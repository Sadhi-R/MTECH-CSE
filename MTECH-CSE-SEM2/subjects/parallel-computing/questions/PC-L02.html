MPI Point-to-Point Communication

High WeightageMid-I Q293%Unit-1Long
Question

Explain blocking and non-blocking MPI send/receive with deadlock scenarios.

DefinitionPoint-to-point communication exchanges data between two specific MPI processes.

Introduction

Blocking waits for operation completion, non-blocking returns immediately.

Why It Is Needed

Choosing correct mode avoids deadlocks and improves overlap.

Working Principle

Use MPI_Send/MPI_Recv or MPI_Isend/MPI_Irecv with Wait.

Step-by-Step Explanation

  • Define modes
  • Show deadlock case
  • Fix using ordering/non-blocking

Architecture / Diagram

Timeline of two processes exchanging messages.

Algorithm

Pairing send/receive operations with tags and ranks.

Flowchart

Post receive -> send data -> wait -> continue.

Real-Time Example

Two bank branches must call and confirm transaction in right order.

Advantages

  • Fine control
  • Supports overlap in non-blocking

Disadvantages

  • Bug-prone if mismatched tags/types
  • Deadlock risk

Applications

  • Domain decomposition
  • Pipeline parallel stages

Exam Tips

Draw matching envelope: source, dest, tag, communicator.

Common Mistakes

  • Using wrong buffer lifetime with non-blocking send.

Difference Table

Non-blocking improves concurrency but needs explicit completion.

Time Complexity

Communication time includes latency + bandwidth term.

Space Complexity

Message buffer space proportional to transferred data.

Interview Questions

  • Interviewers often ask deadlock fix patterns.

Memory Tricks

Post then wait: non-blocking safely overlaps work.
