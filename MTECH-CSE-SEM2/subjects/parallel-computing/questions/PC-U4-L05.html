Question

Explain radix sort.

DefinitionRadix sort sorts integers digit-by-digit (or bit groups) from least significant digit (LSD) or most significant (MSD).

Visual Example

Introduction

Parallel radix sort uses parallel prefix-sum and scatter per digit pass.

Why It Is Needed

Linear time O(d×n) for d digits; excellent for fixed-width integer keys.

Working Principle

For each digit position: parallel count → prefix-sum → scatter elements to stable positions.

Step-by-Step Explanation

  • Determine radix r (e.g., 256 buckets per pass).
  • For each digit d: count keys per bucket.
  • Parallel prefix-sum on counts.
  • Scatter keys to new array by computed offsets.
  • Repeat for all d digit passes.

Architecture / Diagram

Pass1(LSD): count→prefix→scatter
Pass2: repeat on next digit

Algorithm

for digit in 0..d-1: parallel_count → prefix_sum → scatter

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Sort 32-bit ints with r=256: 4 passes, each O(n/p + log p) with prefix sum.

Advantages

  • O(d×n) parallel time
  • Stable sort preserves order
  • GPU-friendly

Disadvantages

  • Extra memory for scratch array
  • Not comparison-based (key type limit)

Applications

  • GPU sorting
  • Database indexing
  • High-throughput analytics

Exam Tips

Show one LSD pass with count/prefix/scatter — core exam content.

Common Mistakes

  • Unstable scatter without prefix-sum offsets.

Difference Table

LSD=least digit first | MSD=most digit first

Time Complexity

O(d × (n/p + log p)) per pass

Space Complexity

Count array + scratch buffer

Interview Questions

  • LSD vs MSD radix?

Memory Tricks

Radix = digit passes with count + prefix + scatter
