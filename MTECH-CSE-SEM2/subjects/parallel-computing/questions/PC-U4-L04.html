Explain bucket and enumeration sort.

Question Bank90%Unit-4Long
Question

Explain bucket and enumeration sort.

DefinitionBucket sort distributes elements into buckets by key range; enumeration sort counts occurrences to determine positions.

Visual Example

Introduction

Both are parallel-friendly when key range is manageable.

Why It Is Needed

Achieve O(n) or O(n+p) parallel time under assumptions on key distribution.

Working Principle

Bucket: hash/range map to buckets, sort buckets locally. Enumeration: count keys, prefix-sum for positions.

Step-by-Step Explanation

  • Bucket: divide range into p buckets.
  • Each PE sorts local bucket.
  • Concatenate buckets (in-order).
  • Enumeration: count each key value.
  • Prefix-sum counts → write to final position.

Architecture / Diagram

Keys→[B0][B1][B2]→local sort→merge in order

Algorithm

Bucket: distribute → local sort → gather
Enumeration: count → prefix sum → scatter

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Sort 1M integers 0-999: 1000 buckets, each PE sorts ~1000 elements.

Advantages

  • Bucket: good uniform distribution
  • Enumeration: O(n) for small key range
  • Highly parallel local sorts

Disadvantages

  • Bucket: bad if skewed distribution
  • Enumeration: range must be small
  • Extra memory for buckets

Applications

  • Radix sort building block
  • Parallel integer sort
  • GPU sorting

Exam Tips

Contrast bucket (range split) vs enumeration (counting) — comparison question.

Common Mistakes

  • Bucket sort when range is huge and sparse.

Difference Table

Bucket=range bins | Enumeration=count keys

Time Complexity

O(n/p) local sort + O(p) merge (bucket)

Space Complexity

Bucket arrays + count arrays

Interview Questions

  • Bucket vs radix sort?

Memory Tricks

Bucket + enumeration = distribute/count then sort

5-Mark Answer

Both algorithms. Steps. Parallel phases. When applicable.

10-Mark Answer

Bucket sort (range bins) and enumeration sort (count + prefix sum) for parallel integer sort.

One-Line RevisionBucket sort partitions by range; enumeration sort uses counting and prefix-sum for positions.
