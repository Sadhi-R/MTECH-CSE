Question

Explain parallel algorithm design.

DefinitionParallel algorithm design transforms sequential solutions into efficient multi-processor versions considering decomposition, communication, and scalability.

Visual Example

Introduction

Design steps: identify parallelism, partition, map, communicate, synchronize, balance load.

Why It Is Needed

Good design achieves iso-efficiency or near-linear speedup.

Working Principle

Analyze dependencies → expose concurrent tasks → minimize critical path and communication volume.

Step-by-Step Explanation

  • Understand sequential algorithm and bottlenecks.
  • Identify task/data parallelism.
  • Choose decomposition (block, recursive).
  • Minimize communication on critical path.
  • Prove/estimate speedup and scalability.

Architecture / Diagram

Sequential → expose parallel tasks → assign to PEs → sync → combine

Algorithm

PCAM: Partitioning, Communication, Agglomeration, Mapping

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Parallel merge sort: two halves sorted concurrently, then merge.

Advantages

  • Systematic PCAM approach
  • Scalability analysis built-in
  • Reusable patterns

Disadvantages

  • Design time longer than sequential
  • Wrong decomposition hurts badly

Applications

  • Sorting
  • Graph algorithms
  • Linear algebra
  • Search

Exam Tips

Mention PCAM four steps — standard exam framework.

Common Mistakes

  • Parallelizing without analyzing communication cost.

Difference Table

Data parallel=same op partitions | Task parallel=different tasks

Time Complexity

T_parallel = T_compute/p + T_comm + T_sync

Space Complexity

Per-PE partition + message buffers
