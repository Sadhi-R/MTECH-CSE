MPI Collective Operations

High WeightageQuestion Bank92%Unit-2Long
Question

Explain MPI_Bcast, MPI_Reduce, MPI_Scatter, MPI_Gather with use cases.

DefinitionCollectives are group communication operations over a communicator.

Introduction

They simplify common patterns and are often optimized internally.

Why It Is Needed

Better than manual point-to-point loops for clarity and speed.

Working Principle

One call executes synchronized data movement pattern across all ranks.

Step-by-Step Explanation

  • Define each collective
  • Give syntax idea
  • Map to practical pattern

Architecture / Diagram

Root process distributing and collecting data among ranks.

Algorithm

Use Bcast/Scatter for distribution; Reduce/Gather for aggregation.

Flowchart

Root prepares data -> collective call -> all ranks receive/process.

Real-Time Example

Teacher shares assignment sheet to all students using Bcast.

Advantages

  • Cleaner code
  • Vendor-optimized implementations

Disadvantages

  • All ranks must participate
  • Synchronization overhead

Applications

  • Parallel sums
  • Distributed training
  • Scientific reductions

Exam Tips

Mention root parameter where applicable.

Common Mistakes

  • Calling collective in only subset of processes.

Difference Table

Reduce gives single aggregate; Allreduce gives everyone aggregate.

Time Complexity

Typically O(log p) style tree communication.

Space Complexity

Temporary buffers as per operation.

Interview Questions

  • Very common in MPI interviews.

Memory Tricks

Collectives are team-level communication shortcuts.
