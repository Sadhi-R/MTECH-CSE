Question

Explain parallel programming platforms.

DefinitionParallel programming platforms provide APIs, runtime, and tools: OpenMP, MPI, CUDA, Hadoop/Spark, and PGAS (UPC, Chapel).

Visual Example

Introduction

Platform choice depends on hardware, scale, and problem structure.

Why It Is Needed

Correct platform reduces development effort and improves performance.

Working Principle

Layer: hardware → runtime (MPI/OpenMP/CUDA) → compiler → application code.

Step-by-Step Explanation

  • OpenMP: shared-memory thread parallelism.
  • MPI: distributed message passing.
  • CUDA: GPU SIMT kernels.
  • Spark: data-parallel analytics on clusters.
  • Compare scope and typical deployment.

Architecture / Diagram

App → OpenMP/MPI/CUDA → OS → Hardware

Algorithm

Match platform to memory model and scale

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

OpenMP on 16-core server; MPI on 256-node cluster; CUDA for GPU training.

Advantages

  • OpenMP: easy loop parallel
  • MPI: portable scale-out
  • CUDA: massive DLP

Disadvantages

  • Mixing models adds complexity
  • Wrong platform wastes hardware

Applications

  • Scientific HPC
  • Big data
  • Deep learning
  • Cloud batch jobs

Common Mistakes

  • Using OpenMP alone on multi-node cluster without shared memory.

Difference Table
