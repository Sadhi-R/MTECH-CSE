Question

Explain parallel matrix-vector multiplication.

DefinitionParallel matrix-vector multiplication computes y = A×x by distributing rows of A across processors.

Visual Example

Introduction

Each processor computes partial dot products for its row subset; no communication if x is replicated.

Why It Is Needed

Foundation for iterative solvers (CG, GMRES) in HPC.

Working Principle

Replicate vector x on all PEs; partition rows of A; local multiply-add; result y is distributed.

Step-by-Step Explanation

  • Replicate x (Broadcast if needed).
  • Partition A by row blocks among p PEs.
  • Each PE: y_local[i] = dot(A_row[i], x).
  • No reduction needed if y partitioned same as rows.
  • Complexity: O(n²/p) compute per PE.

Architecture / Diagram

A rows split: P0→rows0-2, P1→rows3-5
Each: y[i]=Σ A[i][j]*x[j]

Algorithm

Broadcast x → parallel row-wise dot products → distributed y

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

1000×1000 matvec on 4 MPI ranks: 250 rows each, x broadcast once.

Advantages

  • Simple decomposition
  • Minimal communication (one Bcast)
  • Embarrassingly parallel compute

Disadvantages

  • Replicating x uses memory
  • Row split may load-imbalance if sparse

Applications

  • Linear solvers
  • Neural network forward pass
  • PageRank iterations

Exam Tips

State comm: one Broadcast of x; compute O(n²/p).

Common Mistakes

  • Forgetting to broadcast x in distributed setting.

Difference Table

Matvec=row partition | Matmat=2D block better

Time Complexity

O(n²/p) compute + O(n) broadcast

Space Complexity

A partition + replicated x

Interview Questions

  • Why replicate x?

Memory Tricks

Parallel matvec = row split + dot products

5-Mark Answer

Partition scheme. Broadcast x. Per-row compute. Complexity.

10-Mark Answer

Parallel matvec: row-block A, broadcast x, local dot products, O(n²/p) compute.
