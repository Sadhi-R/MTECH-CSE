Question

Explain BFS algorithm.

DefinitionBFS (Breadth-First Search) explores graph level-by-level from source using a queue.

Visual Example

Introduction

Finds shortest path in unweighted graphs; visits all nodes at distance d before d+1.

Why It Is Needed

Parallel BFS uses frontier expansion level-synchronously.

Working Principle

Enqueue source; dequeue vertex, enqueue unvisited neighbors; repeat until queue empty.

Step-by-Step Explanation

  • Mark source s visited, enqueue s.
  • While queue not empty: dequeue v.
  • For each neighbor u of v: if unvisited, mark and enqueue u.
  • Track distance level.
  • Parallel: expand frontier each level.

Architecture / Diagram

Level0:{S} → Level1:{A,B} → Level2:{C,D}

Algorithm

BFS(s): Q=[s]; while Q: v=Q.dequeue(); for u in adj[v]: if !vis[u]: vis[u]=true; Q.enqueue(u)

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Shortest hops in social network: BFS from Alice finds friends at distance 1,2,3.

Advantages

  • Shortest path unweighted
  • O(V+E)
  • Natural level parallelism

Disadvantages

  • O(V) queue memory
  • Not for weighted graphs without modification

Applications

  • GPS unweighted routing
  • Social networks
  • Web crawling
  • Parallel graph analytics

Exam Tips

Show level-by-level expansion diagram.

Common Mistakes

  • Using DFS when shortest path needed.

Difference Table

BFS=shortest unweighted | DFS=deep probe

Time Complexity

O(V+E) sequential; parallel O(diameter) supersteps

Space Complexity

O(V) queue + distance array

Memory Tricks

BFS = queue, level-by-level, shortest path
