Compare mutex and semaphore.

Question Bank90%Unit-3Long
Question

Compare mutex and semaphore.

DefinitionMutex (mutual exclusion lock) allows one thread at a time; semaphore counts permits allowing N concurrent accessors.

Visual Example

Introduction

Mutex is binary; semaphore is counting synchronization primitive.

Why It Is Needed

Choose based on resource count and synchronization need.

Working Principle

Mutex lock before critical section; semaphore wait (P) decrements count, signal (V) increments.

Step-by-Step Explanation

  • Mutex: lock/unlock, one owner.
  • Binary semaphore: like mutex (0/1).
  • Counting semaphore: N slots (pool).
  • Mutex: ownership required.
  • Semaphore: no ownership, signal by any thread.

Architecture / Diagram

Mutex: [====lock====] one thread
Semaphore(3): up to 3 threads in pool

Algorithm

Mutex: pthread_mutex_lock → CS → unlock
Sem: sem_wait → use resource → sem_post

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Mutex protects bank balance update; semaphore limits 5 concurrent DB connections.

Advantages

  • Mutex: simple exclusion
  • Semaphore: resource counting
  • Both prevent races

Disadvantages

  • Mutex deadlock if circular wait
  • Semaphore harder to debug
  • Priority inversion possible

Applications

  • Critical sections
  • Connection pools
  • Producer-consumer bounded buffer

Exam Tips

Table comparing ownership, count, and use case — exam classic.

Common Mistakes

  • Using mutex when counting semaphore needed (fixed pool size).

Difference Table

Mutex=1 at a time, owner | Semaphore=N permits, no owner

Time Complexity

Uncontended lock O(1); contention serializes

Space Complexity

Lock object + wait queue

Interview Questions

  • Binary vs counting semaphore?

Memory Tricks

Mutex=exclusive lock; Semaphore=counted permits
