Parallel Load Balancing

High WeightageQuestion Bank91%Unit-3Long
Question

Discuss static and dynamic load balancing techniques in parallel computing.

DefinitionLoad balancing distributes work so processors finish around the same time.

Introduction

Imbalance causes idle processors and poor speedup.

Why It Is Needed

Critical for scalability in irregular workloads.

Working Principle

Use partitioning and runtime work redistribution methods.

Step-by-Step Explanation

  • Define imbalance metric
  • Static strategies
  • Dynamic strategies
  • Trade-offs

Architecture / Diagram

Gantt chart showing balanced vs imbalanced processor timelines.

Algorithm

Work stealing and chunk scheduling algorithms.

Flowchart

Partition -> monitor -> migrate/steal tasks -> complete.

Real-Time Example

Restaurant orders dynamically moved to less busy chefs.

Advantages

  • Improves utilization
  • Reduces total completion time

Disadvantages

  • Migration overhead
  • Complex runtime design

Applications

  • Graph processing
  • Simulation
  • Task schedulers

Exam Tips

Mention overhead vs balance trade-off.

Common Mistakes

  • Assuming equal data size implies equal compute time always.

Difference Table

Static low overhead; dynamic adapts better to unpredictability.

Time Complexity

Balancing lowers effective critical path time.

Space Complexity

Extra queues/metadata for task management.

Interview Questions

  • Interviewers ask static vs dynamic scenarios.

Memory Tricks

Keep all workers equally busy.

5-Mark Answer

Define both methods and when to use.

10-Mark Answer

Explain metrics, algorithms, and practical impact.
