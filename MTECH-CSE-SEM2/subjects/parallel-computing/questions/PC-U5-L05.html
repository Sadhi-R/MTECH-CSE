Question

Compare DFS and BFS.

DefinitionBFS uses a Queue (level-by-level). DFS uses a Stack/recursion (depth-first, backtrack).

Visual Example

Visual Example

Introduction

Graph traversal visits all reachable nodes systematically.

Why It Is Needed

Needed for shortest paths, cycle detection, connectivity, topological sort.

Working Principle

BFS: enqueue source, dequeue, enqueue unvisited neighbors. DFS: mark visited, recurse on neighbors, backtrack.

Step-by-Step Explanation

  • Draw graph.
  • Show BFS order with queue contents.
  • Show DFS order with stack.
  • Write pseudocode for both.
  • State O(V+E) complexity.

Architecture / Diagram

  A---B---D
   \     /
    C---E
BFS from A: A→B,C→D,E
DFS from A: A→B→D→E→C

Algorithm

BFS: queue-based. DFS: recursive or explicit stack.

Flowchart

Start → mark source → BFS: enqueue loop | DFS: recurse unvisited → done

Real-Time Example

BFS = Google Maps equal roads. DFS = maze / friend-chain exploration.

Advantages

  • BFS: shortest path unweighted
  • DFS: less memory on deep graphs, topo sort

Disadvantages

  • BFS: more memory on wide graphs
  • DFS: may not find shortest path

Applications

  • BFS: GPS hops, broadcasting
  • DFS: cycle detect, topo sort, SCC

Exam Tips

Always state data structure: Queue vs Stack. Time O(V+E) for both.

Common Mistakes

  • Using Stack for BFS
  • forgetting visited array.

Difference Table

BFS=Queue+levels+shortest | DFS=Stack+depth+backtrack

Time Complexity

O(V+E) both

Space Complexity

O(V) visited + queue/stack

Interview Questions

  • When BFS over DFS? How detect cycle with DFS?

Memory Tricks

BFS=Big Queue, DFS=Dive Stack
