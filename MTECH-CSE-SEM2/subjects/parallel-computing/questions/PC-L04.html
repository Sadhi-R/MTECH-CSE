PThreads and Synchronization

High WeightageMid-I Q394%Unit-2Long
Question

Explain PThreads model with mutexes, condition variables, and race conditions.

DefinitionPThreads is a POSIX API for creating and managing threads in shared memory.

Introduction

Threads share address space, so synchronization is required.

Why It Is Needed

Without synchronization, race conditions produce wrong results.

Working Principle

Use mutex lock/unlock and condition wait/signal around shared data.

Step-by-Step Explanation

  • Create threads
  • Share data
  • Protect critical section
  • Join threads

Architecture / Diagram

Threads competing for shared counter with lock protection.

Algorithm

Thread function plus mutex-protected update pattern.

Flowchart

Spawn threads -> lock -> update -> unlock -> join.

Real-Time Example

Restaurant billing desk where one cashier updates ledger at a time.

Advantages

  • Lightweight parallelism
  • Good for multicore CPUs

Disadvantages

  • Deadlocks possible
  • Hard debugging for races

Applications

  • Web servers
  • Image processing
  • Parallel loops

Exam Tips

Write one race-condition example then fixed version.

Common Mistakes

  • Holding lock during long I/O causing contention.

Difference Table

MPI isolates memory; PThreads shares memory directly.

Time Complexity

Depends on thread count and lock contention.

Space Complexity

Shared data plus thread stacks.

Interview Questions

  • Frequently asked for systems developer roles.

Memory Tricks

Lock before touch shared state.

5-Mark Answer

Explain thread lifecycle and synchronization primitives.

10-Mark Answer

Include race, mutex, condition variable, and deadlock avoidance.
