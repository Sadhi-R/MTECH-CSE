MPI Programming Model

High WeightageMid-I Q195%Unit-1Long
Question

Explain MPI model, communicator, rank, and basic program structure.

DefinitionMPI is a message-passing standard for distributed-memory parallel systems.

Introduction

Each process has private memory and communicates explicitly.

Why It Is Needed

It is the backbone of many HPC applications.

Working Principle

Program starts all processes, each gets rank, then exchanges messages.

Step-by-Step Explanation

  • MPI_Init
  • Get rank/size
  • Compute local work
  • Communicate
  • MPI_Finalize

Architecture / Diagram

Multiple processes connected by network and communicator.

Algorithm

SPMD style pseudo algorithm with send/recv/collectives.

Flowchart

Start -> init -> local compute -> communication -> finalize.

Real-Time Example

Delivery centers in different cities exchange shipment updates.

Advantages

  • Portable standard
  • Scales across clusters

Disadvantages

  • Communication overhead
  • Debugging complexity

Applications

  • Weather simulation
  • CFD
  • Large matrix operations

Exam Tips

Always mention communicator and rank concept.

Common Mistakes

  • Forgetting matching send and receive arguments.

Difference Table

MPI is process-based; PThreads is shared-memory thread-based.

Time Complexity

Depends on algorithm and communication pattern.

Space Complexity

Each process has local memory plus message buffers.

Interview Questions

  • Common in systems/HPC interviews.

Memory Tricks

MPI = many separate workers talking by messages.
