Question

Explain parallel matrix-matrix multiplication.

DefinitionParallel matrix-matrix multiplication uses block decomposition (Cannon's, SUMMA, or 2D block) to minimize communication.

Visual Example

Introduction

Naive 1D row-column split causes O(n²) communication; 2D block achieves O(n²/√p).

Why It Is Needed

Core kernel in BLAS Level 3 and deep learning.

Working Principle

Partition A, B, C into p blocks; each PE computes local block products with neighbor exchanges.

Step-by-Step Explanation

  • 2D block partition: √p × √p grid.
  • Initial alignment shuffle (Cannon).
  • Local GEMM on owned blocks.
  • Shift A rows, B columns each step.
  • Accumulate into local C block.

Architecture / Diagram

2×2 process grid:
P00: C00+=A00B00+A01B10 ...

Algorithm

Cannon: align → k iterations local mul + shift → complete C

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

1024×1024 matmul on 16 processors (4×4 grid): each owns 256×256 blocks.

Advantages

  • 2D block optimal comm
  • Scalable Cannon/SUMMA
  • High compute/comm ratio

Disadvantages

  • Complex implementation
  • Non-square p needs padding

Applications

  • BLAS GEMM
  • Deep learning training
  • Scientific simulation

Exam Tips

Compare 1D O(n²) vs 2D O(n²/√p) comm — key exam point.

Common Mistakes

  • 1D decomposition on large p (communication dominates).

Difference Table

1D row split=simple high comm | 2D block=optimal

Time Complexity

O(n³/p) compute + O(n²/√p) comm (2D)

Space Complexity

Local A,B,C blocks per PE

Interview Questions

  • Cannon vs SUMMA?

Memory Tricks

Parallel matmul = 2D block + Cannon/SUMMA

5-Mark Answer

Decompositions. Cannon steps. Comm volume. Example.

10-Mark Answer

Parallel matmul via 2D block decomposition (Cannon/SUMMA) minimizing communication.
