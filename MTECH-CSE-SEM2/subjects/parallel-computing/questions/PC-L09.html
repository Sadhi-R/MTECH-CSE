Communication Cost Models

High WeightageQuestion Bank90%Unit-5Long
Question

Explain latency-bandwidth model for communication in parallel systems.

DefinitionMessage time is modeled as T = alpha + beta * m where m is message size.

Introduction

Alpha is startup latency, beta is per-byte transfer cost.

Why It Is Needed

Model guides algorithm design and message aggregation decisions.

Working Principle

Estimate total communication from number and size of messages.

Step-by-Step Explanation

  • Define alpha and beta
  • Compute sample
  • Discuss optimization methods

Architecture / Diagram

Line graph of message time vs size with fixed intercept.

Algorithm

Cost estimation algorithm for collective communication phases.

Flowchart

Count messages -> estimate each cost -> sum.

Real-Time Example

Courier booking has fixed pickup time plus per-kilometer cost.

Advantages

  • Simple predictive model
  • Useful in comparing designs

Disadvantages

  • Ignores contention and topology details

Applications

  • MPI tuning
  • Cluster planning
  • Algorithm analysis

Exam Tips

Use units clearly (microseconds and bytes).

Common Mistakes

  • Mixing latency and bandwidth dimensions.

Difference Table

Few large messages often better than many tiny messages.

Time Complexity

Model evaluation O(number of message events).

Space Complexity

O(1) per event for estimation.

Interview Questions

  • Very common in HPC interview discussions.

Memory Tricks

Communication has fixed startup plus size-dependent cost.

5-Mark Answer

Write formula and one calculation.

10-Mark Answer

Apply model to compare two communication strategies.
