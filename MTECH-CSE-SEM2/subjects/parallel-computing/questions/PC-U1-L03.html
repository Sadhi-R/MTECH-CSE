Question

Explain shared versus distributed memory architecture.

DefinitionShared memory provides global address space to all threads; distributed memory gives each process private memory requiring explicit messages.

Visual Example

Introduction

Shared: OpenMP/pthreads on SMP/NUMA. Distributed: MPI on clusters.

Why It Is Needed

Fundamental split in parallel programming models.

Working Principle

Shared: load/store to common variables (with sync). Distributed: Send/Recv to exchange data.

Step-by-Step Explanation

  • Shared: threads, global vars, locks.
  • Distributed: processes, no shared address.
  • NUMA: shared but non-uniform latency.
  • Programming: OpenMP vs MPI.
  • Hybrid: MPI+OpenMP on clusters.

Architecture / Diagram

Shared: [T1 T2 T3]→Global RAM
Distributed: [P1+Mem]—net—[P2+Mem]

Algorithm

Identify coupling → choose shared (low comm) or distributed (scale out)

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Multicore array sum: shared counter with lock. Cluster sort: MPI Send/Recv partitions.

Advantages

  • Shared: easy data access
  • Distributed: massive scale
  • Hybrid combines both

Disadvantages

  • Shared: race conditions
  • Distributed: explicit comm coding
  • Shared doesn't scale infinitely

Applications

  • Multicore desktop apps
  • HPC clusters
  • Cloud microservices

Exam Tips

Diagram both models side-by-side — essential for comparison questions.

Common Mistakes

  • Assuming shared memory on MPI cluster without shared-memory extensions.

Difference Table

Shared=global address | Distributed=message passing

Time Complexity

Shared: mem latency | Distributed: network latency

Space Complexity

Shared: one address space | Distributed: per-process heap

Interview Questions

  • NUMA vs UMA?

Memory Tricks

Shared=OpenMP global; Distributed=MPI messages
