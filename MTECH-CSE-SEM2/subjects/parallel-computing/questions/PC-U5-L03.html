Question

Explain DFS algorithm.

DefinitionDFS (Depth-First Search) explores graph by going as deep as possible before backtracking, using stack/recursion.

Visual Example

Introduction

Visits vertices in depth order; fundamental for connectivity, cycles, and topological sort.

Why It Is Needed

Parallel DFS is harder due to sequential stack nature; used in decomposition.

Working Principle

Mark visited; recursively visit unvisited neighbors; backtrack when dead end.

Step-by-Step Explanation

  • Mark all vertices unvisited.
  • For each unvisited v: DFS(v).
  • DFS(v): mark visited; for each neighbor u unvisited: DFS(u).
  • Use stack for iterative version.
  • Record discovery/finish times.

Architecture / Diagram

A→B→D→back→C
Stack: push A, push B, push D, pop D, pop B...

Algorithm

DFS(v): visited[v]=true; for u in adj[v]: if !visited[u]: DFS(u)

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Detect cycle in course prereqs: DFS back edge to ancestor on stack.

Advantages

  • O(V+E) time
  • O(V) space
  • Simple recursive structure

Disadvantages

  • Deep graphs risk stack overflow
  • Not shortest path

Applications

  • Cycle detection
  • Topological sort
  • Maze solving
  • Connected components

Exam Tips

Show discovery/finish times on small DAG.

Common Mistakes

  • Not marking visited → infinite loop on cycles.

Difference Table

DFS=depth first stack | BFS=level order queue

Time Complexity

O(V+E)

Space Complexity

O(V) visited + stack

Interview Questions

  • DFS vs BFS when?

Memory Tricks

DFS = go deep, backtrack, use stack

5-Mark Answer

Algorithm. Recursive/iterative. Times. Cycle detection.

10-Mark Answer

DFS: stack/recursion, O(V+E), applications in cycles and topo sort.
