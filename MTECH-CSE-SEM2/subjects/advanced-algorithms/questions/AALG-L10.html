NP-Completeness and Reductions

High WeightageMid-II Q396%Unit-5Long
Question

Define P, NP, NP-hard, NP-complete and explain polynomial reduction process.

DefinitionNP-complete problems are in NP and at least as hard as every NP problem.

Introduction

Reductions transfer hardness from known NP-complete problems.

Why It Is Needed

This classification guides realistic algorithm expectations.

Working Principle

To prove X NP-complete: show X in NP, reduce known NP-complete Y to X.

Step-by-Step Explanation

  • Define classes
  • Explain verifier idea
  • Describe reduction
  • Conclude hardness

Architecture / Diagram

Set relation diagram: P subset NP, NP-complete intersection.

Algorithm

No runtime algorithm; proof strategy algorithmic transformation.

Flowchart

Known NPC problem -> polynomial transform -> target problem.

Real-Time Example

Converting scheduling constraints to graph coloring instance.

Advantages

  • Prevents wasted search for exact poly-time solution
  • Supports approximation focus

Disadvantages

  • Proof writing can be abstract
  • No direct constructive solver

Applications

  • Compiler optimization
  • Logistics planning
  • Security

Exam Tips

Name source problem explicitly (SAT/3SAT/CLIQUE).

Common Mistakes

  • Claiming NP-hard implies NP-complete without NP membership proof.

Difference Table

NP-hard may lie outside NP; NP-complete is inside NP.

Time Complexity

Reduction itself polynomial time.

Space Complexity

Space follows transformation size.

Interview Questions

  • Very common in written and oral interviews.

Memory Tricks

If SAT can become your problem, your problem is likely hard.

5-Mark Answer

Write two-step proof template.

10-Mark Answer

Give complete reduction outline with correctness argument.
