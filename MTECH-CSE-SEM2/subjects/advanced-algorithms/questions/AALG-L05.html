Bipartite Matching via Flow

Question Bank90%Unit-3Long
Question

How do you convert maximum bipartite matching to max-flow problem?

DefinitionBipartite matching pairs vertices from two disjoint sets without conflict.

Introduction

A standard reduction maps matching to unit-capacity flow.

Why It Is Needed

Reduction simplifies proof and implementation using known algorithms.

Working Principle

Add source to left set, right set to sink, capacities 1 on all edges.

Step-by-Step Explanation

  • Create directed network
  • Run max-flow
  • Read matched edges with flow 1

Architecture / Diagram

Left nodes and right nodes with source/sink layers.

Algorithm

Use max-flow algorithm on transformed graph.

Flowchart

Build network -> run flow -> extract pairs.

Real-Time Example

Assigning interview slots (students to panels) without overlaps.

Advantages

  • Clear correctness
  • Polynomial-time solution

Disadvantages

  • Graph size increases
  • Need integral capacities

Applications

  • Job assignment
  • Mentor-mentee pairing
  • Resource allocation

Exam Tips

Mention integrality theorem for integer capacities.

Common Mistakes

  • Allowing multiple matches for one vertex by wrong capacities.

Difference Table

Hungarian solves weighted matching; this handles unweighted maximum cardinality.

Time Complexity

Using Dinic often near O(E*sqrt(V)) in bipartite case.

Space Complexity

Linear in transformed graph size.

Interview Questions

  • Popular theory-to-practice reduction question.

Memory Tricks

One person, one seat, capacity one.

5-Mark Answer

Draw transformed network clearly.

10-Mark Answer

Show reduction proof and extract final matching.
