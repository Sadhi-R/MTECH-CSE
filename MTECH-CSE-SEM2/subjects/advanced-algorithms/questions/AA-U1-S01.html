Why does BFS give shortest path in an unweighted graph?

DefinitionBFS (Breadth-First Search) visits a graph level by level using a Queue.

Visual Example

Key Points

  • BFS explores nodes in increasing distance (number of edges) from the source.
  • The first time BFS reaches a node, that path uses the minimum number of edges.
  • Therefore in an unweighted graph (all edges cost 1), BFS gives the shortest path.

Real-Time Example

Google Maps treating all roads as equal distance — BFS finds the route with fewest turns/hops.

Exam Keyword Queue · Level-order · O(V+E) · Unweighted graph
One-Line RevisionBFS = Queue + level-by-level → first visit = shortest path in unweighted graphs.
