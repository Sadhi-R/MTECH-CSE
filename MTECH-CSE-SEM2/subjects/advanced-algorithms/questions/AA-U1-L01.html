Compare five sorting algorithms.

Question Bank90%Unit-1Long
Question

Compare five sorting algorithms.

DefinitionSorting arranges elements in order. Different algorithms trade speed, memory, and stability.

Visual Example

Introduction

Comparison sorts are fundamental to databases, search, and scheduling.

Why It Is Needed

Choosing the right sort affects performance on large datasets (Amazon orders, exam roll lists).

Working Principle

Compare/swap or divide-and-conquer to place elements correctly.

Step-by-Step Explanation

  • State purpose of sorting.
  • Tabulate Bubble, Insertion, Selection, Merge, Quick.
  • Give best/avg/worst time and space.
  • Note stability.
  • Give one advantage and disadvantage each.

Architecture / Diagram

Bubble: adjacent swaps | Merge: split-merge | Quick: partition around pivot

Algorithm

Merge Sort: divide array, sort halves, merge. Quick Sort: pick pivot, partition, recurse.

Flowchart

Choose sort → Compare n and data pattern → Pick Merge/Quick for large n

Real-Time Example

Sorting delivery packages by pin code before dispatch.

Advantages

  • Merge/Quick O(n log n) average
  • Insertion good for nearly sorted

Disadvantages

  • Bubble/Selection O(n²)
  • Quick worst case O(n²) bad pivot

Applications

  • Database indexing
  • Finding median
  • Preprocessing for binary search

Exam Tips

Always include comparison table with stability column.

Common Mistakes

  • Saying Quick is always O(n log n)
  • forgetting Merge needs O(n) extra space.

Difference Table

BubbleO(n²)O(1)Yes
MergeO(n log n)O(n)Yes
QuickO(n log n) avgO(log n)No

Time Complexity

Bubble O(n²), Merge/Quick O(n log n) average

Space Complexity

Merge O(n), others mostly O(1) or O(log n)

Interview Questions

  • When to use stable sort? When is Insertion Sort preferred?

Memory Tricks

Bubble = Buddy swaps; Merge = Memory needed; Quick = Quick if good pivot

5-Mark Answer

Define sorting. Tabulate 5 algorithms with time/space/stability. One line each advantage.

10-Mark Answer

Full table + explain Merge and Quick with small example dry-run + when to use which.

One-Line RevisionMerge/Quick for large data; Bubble/Insertion for teaching and tiny/nearly-sorted data.
