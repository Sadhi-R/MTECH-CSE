Question

Explain Warshall technique.

DefinitionWarshall's algorithm computes the transitive closure of a directed graph — whether path exists from every i to every j.

Visual Example

Introduction

Warshall uses dynamic programming similar to Floyd-Warshall but for reachability (boolean).

Why It Is Needed

Needed to know if one node can reach another through any path (dependency chains, reachability).

Working Principle

For each intermediate k, set reach[i][j] = reach[i][j] OR (reach[i][k] AND reach[k][j]).

Step-by-Step Explanation

  • Build n×n boolean matrix from adjacency.
  • Initialize: reach[i][j]=1 if edge i→j or i=j.
  • Loop k from 1 to n.
  • Loop i,j: reach[i][j] |= reach[i][k] & reach[k][j].
  • Final matrix shows all reachable pairs.

Architecture / Diagram

Graph: 1→2, 2→3
Reach matrix after Warshall: 1 reaches 2,3; 2 reaches 3

Algorithm

for k=1..n:
  for i=1..n:
    for j=1..n:
      R[i][j] = R[i][j] OR (R[i][k] AND R[k][j])

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Software module dependencies — Warshall finds which modules transitively depend on others.

Advantages

  • Simple triple loop
  • Works for all pairs
  • O(V³) predictable

Disadvantages

  • O(V³) slow for huge graphs
  • Only reachability, not shortest path

Applications

  • Transitive closure
  • Dependency analysis
  • Compiler dead-code detection

Exam Tips

Contrast with Floyd-Warshall: Warshall=boolean reachability; Floyd=shortest distances.

Common Mistakes

  • Using Warshall when you need shortest path lengths (use Floyd-Warshall).

Difference Table

Warshall=reachability boolean | Floyd=shortest distance numeric

Time Complexity

O(V³)

Space Complexity

O(V²) matrix
