Question

Explain dynamic programming and Floyd-Warshall with limitations.

DefinitionFloyd-Warshall finds all-pairs shortest paths using dynamic programming in O(V³).

Visual Example

Introduction

DP tries every intermediate vertex k to improve paths.

Why It Is Needed

When you need distances between every pair of nodes (flight matrix).

Working Principle

For each k,i,j: dist[i][j] = min(dist[i][j], dist[i][k]+dist[k][j]).

Step-by-Step Explanation

  • Initialize dist from adjacency matrix (∞ if no edge, 0 diagonal).
  • Loop k from 1 to V.
  • Loop i,j: try k as intermediate.
  • Output dist matrix.

Architecture / Diagram

3 nested loops over k, i, j

Algorithm

Triple loop DP. Path reconstruction via predecessor matrix optional.

Flowchart

For each intermediate k → update all pairs

Real-Time Example

Cheapest flight between every pair of cities in an airline network.

Advantages

  • Simple to code
  • Handles negative weights (no negative cycles)
  • All-pairs in one run

Disadvantages

  • O(V³) slow for large V
  • Fails with negative cycles
  • No path info without extra matrix

Applications

  • Transitive closure
  • All-pairs routing
  • Game map analysis

Exam Tips

State limitation: negative cycles. Paradigm: Dynamic Programming.

Common Mistakes

  • Using for single-source only (use Dijkstra instead).

Difference Table

Dijkstra=single-source O((V+E)logV) | Floyd=all-pairs O(V³)

Time Complexity

O(V³)

Space Complexity

O(V²)

Interview Questions

  • When Floyd vs running Dijkstra V times?

Memory Tricks

Floyd = For All pairs, 3 nested loops
