Question

Explain airline scheduling using flow networks.

DefinitionAirline scheduling maps to max-flow: maximize passengers routed from sources to destinations through flight legs (edges with capacity).

Visual Example

Introduction

Flow networks model capacity-constrained routing.

Why It Is Needed

Airlines want maximum revenue flights within aircraft/seat capacity limits.

Working Principle

Vertices = airports; edges = flights with seat capacity; source = hub, sink = destinations; max-flow = max passengers.

Step-by-Step Explanation

  • Build flow network: airports, flight capacities.
  • Add supersource/super-sink if multiple origins/destinations.
  • Run Edmonds-Karp (BFS augmenting paths).
  • Flow value = max schedulable passengers.
  • Min-cut identifies bottleneck routes.

Architecture / Diagram

Hub→CityA(cap200)→CityB(cap150)→Sink; max flow limited by min cut

Algorithm

Edmonds-Karp on flight network; augment until no s-t path in residual graph

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Delhi hub connecting 10 cities — max-flow finds maximum passenger throughput.

Advantages

  • Models real capacity limits
  • Polynomial algorithm
  • Min-cut shows bottlenecks

Disadvantages

  • Simplified model ignores time windows
  • Integer capacities assumed

Applications

  • Airline scheduling
  • Railway routing
  • Supply chain

Exam Tips

Draw network with capacities; show augmenting path.

Common Mistakes

  • Not including reverse edges in residual graph.

Difference Table

Max-flow=max throughput | Shortest path=min distance

Time Complexity

Edmonds-Karp O(V E²)

Space Complexity

O(V+E) residual graph
