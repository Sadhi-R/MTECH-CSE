Question

Explain preflow-push algorithm.

DefinitionPreflow-push is an advanced max-flow algorithm that pushes excess flow locally and relabels vertices.

Visual Example

Introduction

More efficient than Edmonds-Karp for many dense graphs: O(V²E) or O(V³) variants.

Why It Is Needed

Uses height labels and excess at vertices instead of global augmenting paths.

Working Principle

Initialize preflow saturating source edges; repeatedly push flow downhill or relabel when stuck.

Step-by-Step Explanation

  • Initialize: height[s]=n, height[others]=0, saturate source edges.
  • While excess exists: push from u to v if height[u]=height[v]+1.
  • If cannot push: relabel u (increase height).
  • Stop when all excess returns to source/sink.
  • Total flow = sum out of source.

Architecture / Diagram

Push excess along admissible edges (downhill in height); relabel when no admissible edge

Algorithm

while excess[v]>0 and v≠s,t:
  try push to neighbour with lower height
  else relabel v

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Water network: push water from reservoir downhill; relabel when blocked.

Advantages

  • Often faster in practice
  • Local operations
  • Good for dense networks

Disadvantages

  • More complex to implement
  • Worst case still high for some variants

Applications

  • Max-flow
  • Image segmentation
  • Bipartite matching reduction

Exam Tips

Contrast with Ford-Fulkerson: local push vs global augmenting path.

Common Mistakes

  • Pushing to vertex at same height (must be height[u]=height[v]+1).

Difference Table

Preflow-push=local push+relabel | Edmonds-Karp=global BFS augment

Time Complexity

O(V²E) generic; O(V³) with FIFO queue

Space Complexity

O(V) height and excess arrays

Interview Questions

  • Push vs relabel — when each?

Memory Tricks

Preflow = Push excess Downhill, Relabel if stuck
