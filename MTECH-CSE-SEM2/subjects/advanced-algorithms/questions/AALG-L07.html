Floyd-Warshall All Pairs Shortest Path

High WeightageQuestion Bank92%Unit-4Long
Question

Explain Floyd-Warshall dynamic programming algorithm with complexity.

DefinitionFloyd-Warshall computes shortest paths between all vertex pairs.

Introduction

It uses intermediate-vertex based DP transitions.

Why It Is Needed

Useful when distances for many source-destination pairs are needed.

Working Principle

Update d[i][j] = min(d[i][j], d[i][k] + d[k][j]) for each k.

Step-by-Step Explanation

  • Initialize matrix
  • Iterate k
  • Relax every i,j pair
  • Check negative cycle

Architecture / Diagram

Distance matrix updated layer by layer for each k.

Algorithm

Triple nested loops implementing DP transition.

Flowchart

k loop outside -> i,j loops inside -> final matrix.

Real-Time Example

Precomputing travel costs between all warehouse pairs.

Advantages

  • Simple and elegant
  • Handles negative edges

Disadvantages

  • O(n^3) expensive for very large graphs

Applications

  • Routing tables
  • Transitive closure
  • Network analysis

Exam Tips

Mention diagonal negative value indicates negative cycle.

Common Mistakes

  • Wrong loop order that breaks DP dependency.

Difference Table

Compared to repeated Dijkstra: better on dense graphs and many queries.

Time Complexity

O(V^3).

Space Complexity

O(V^2) matrix.

Interview Questions

  • Often asked as direct DP formula question.

Memory Tricks

Allowing each node as middle stop one by one.

5-Mark Answer

State recurrence clearly.

10-Mark Answer

Show matrix evolution for one k value.
