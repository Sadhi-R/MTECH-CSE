Question

Compare chromatic number concept and DAG scheduling.

DefinitionChromatic number χ(G) = minimum colours for proper vertex colouring. DAG scheduling orders tasks with prerequisites (topological sort).

Visual Example

Introduction

Both involve ordering/assignment constraints on graphs.

Why It Is Needed

Chromatic number for resource allocation; DAG scheduling for project/course planning.

Working Principle

Colouring: minimize colours with adjacency constraint. DAG scheduling: linear order respecting edge direction.

Step-by-Step Explanation

  • Define chromatic number with example (cycle C4 needs 2 colours).
  • Define DAG and topological sort.
  • Kahn's algorithm for topo sort.
  • Compare: colouring=parallel slots; topo=sequential order.
  • Both are graph constraint problems.

Architecture / Diagram

Cycle C4: χ=2 (alternate colours)
DAG courses: Math→OS→Cloud → topo order Math,OS,Cloud

Algorithm

Topo sort (Kahn): in-degree 0 queue → dequeue → reduce neighbours

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Factory: colouring = machines that cannot run conflicting jobs simultaneously; DAG = assembly line order.

Advantages

  • Chromatic: models conflicts
  • Topo sort: O(V+E) on DAGs
  • Both graph-based planning

Disadvantages

  • Optimal colouring hard
  • Topo sort fails on cycles

Applications

  • Timetabling vs project scheduling
  • Compiler instruction scheduling

Exam Tips

Draw side-by-side: colouring graph vs DAG with arrows.

Common Mistakes

  • Using topological sort on cyclic graph.

Difference Table

Chromatic=minimum colours | DAG scheduling=valid task order

Time Complexity

Topo sort O(V+E); colouring optimal NP-hard

Space Complexity

O(V) or O(V²)

Interview Questions

  • Can topological sort solve graph colouring?

Memory Tricks

Chromatic=colours; DAG=prerequisite order
