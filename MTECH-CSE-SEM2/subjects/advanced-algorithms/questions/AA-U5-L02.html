Question

Explain random dictionaries.

DefinitionRandom dictionaries (hash tables with randomization) achieve O(1) expected search/insert using universal hashing.

Visual Example

Introduction

Random hash function spreads keys evenly, minimizing collisions.

Why It Is Needed

Deterministic worst-case O(n) for hash tables; randomization gives expected O(1).

Working Principle

Pick random hash from universal family H; expected chain length O(1) for n keys, m slots.

Step-by-Step Explanation

  • Define universal hash family: Pr[h(x)=h(y)]≤1/m for x≠y.
  • Pick random h from H at startup.
  • Insert/search in bucket h(key).
  • Expected O(1) with load factor α=n/m bounded.
  • Rehash if load factor too high.

Architecture / Diagram

m slots, n keys, random h → expected chain length n/m

Algorithm

h(k)=((ak+b) mod p) mod m for random a,b — universal family

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Compiler symbol table uses hash map for fast variable lookup.

Advantages

  • Expected constant time
  • Simple implementation

Disadvantages

  • Worst case still O(n) if unlucky
  • Rehashing cost

Applications

  • Symbol tables
  • Database indexing
  • Caches

Exam Tips

State universal hashing definition — exam definition question.

Common Mistakes

  • Confusing average case with worst case.

Difference Table

Random dict=expected O(1) | Worst case O(n)

Time Complexity

Expected O(1); worst O(n)

Space Complexity

O(n+m)
