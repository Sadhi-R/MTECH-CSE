Question

Write NP-hardness proof strategy.

DefinitionNP-hardness proof strategy: reduce a known NP-complete problem to target problem X in polynomial time.

Visual Example

Introduction

If Y reduces to X and Y is NP-hard, then X is NP-hard.

Why It Is Needed

Proves no known polynomial exact algorithm unless P=NP.

Working Principle

Transform instance of SAT/3-SAT/Vertex Cover to instance of X preserving yes/no answer.

Step-by-Step Explanation

  • Show X in NP (polynomial verifier) if proving NP-complete.
  • Pick known NP-complete Y (usually 3-SAT).
  • Build polynomial transformation f: Y→X.
  • Prove: Y yes ⟺ X yes.
  • Conclude X is NP-hard (or NP-complete).

Architecture / Diagram

3-SAT → Vertex Cover → Your problem X (chain of reductions)

Algorithm

Given φ in 3-SAT, construct graph G such that φ satisfiable ⟺ G has VC of size k

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Prove TSP decision is NP-hard by reducing Hamiltonian Cycle to TSP.

Advantages

  • Standard proof technique
  • Classifies problem difficulty

Disadvantages

  • Does not prove no algorithm — only no poly algorithm known
  • Reduction must be correct both ways

Applications

  • Complexity classification
  • Choosing approximation vs exact

Exam Tips

Cook-Levin: SAT is first NP-complete. Always show both directions of reduction.

Common Mistakes

  • One-way reduction (must prove yes→yes AND no→no).

Difference Table

NP-hard=at least as hard | NP-complete=in NP + NP-hard

Time Complexity

Reduction in poly time

Space Complexity

O(n) transformation size
