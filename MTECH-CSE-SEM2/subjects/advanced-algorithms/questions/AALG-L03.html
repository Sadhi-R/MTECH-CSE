Single Source Shortest Path: Dijkstra

High WeightageMid-I Q294%Unit-1Long
Question

Describe Dijkstra algorithm, correctness idea, and limitations.

DefinitionDijkstra finds shortest distances from one source in non-negative weighted graphs.

Introduction

It repeatedly settles nearest unsettled vertex.

Why It Is Needed

Used in route planners and network latency optimization.

Working Principle

Priority queue picks minimum tentative distance each step.

Step-by-Step Explanation

  • Initialize distances
  • Extract-min vertex
  • Relax outgoing edges
  • Repeat

Architecture / Diagram

Weighted graph with settled and unsettled sets.

Algorithm

Use min-heap and relaxation formula d[v] > d[u] + w(u,v).

Flowchart

Start source -> relax neighbors -> settle min -> finish.

Real-Time Example

Google Maps can use it when no negative weights exist.

Advantages

  • Fast for sparse graphs
  • Simple implementation

Disadvantages

  • Fails with negative edges
  • Needs careful heap handling

Applications

  • Navigation
  • Network routing
  • Game AI paths

Exam Tips

Write relaxation equation exactly.

Common Mistakes

  • Forgetting to skip stale heap entries.

Difference Table

Bellman-Ford handles negatives but slower.

Time Complexity

O((V+E)logV) with binary heap.

Space Complexity

O(V+E).

Interview Questions

  • Frequent coding interview graph problem.

Memory Tricks

Settle nearest city first, then expand.

5-Mark Answer

Core idea and one iteration trace.

10-Mark Answer

Proof sketch using greedy choice and contradiction.
