Question

Explain graph colouring.

DefinitionGraph colouring assigns colours to vertices so no two adjacent vertices share the same colour.

Visual Example

Introduction

Minimum colours needed = chromatic number χ(G). NP-hard to compute exactly.

Why It Is Needed

Used in timetabling, register allocation, frequency assignment.

Working Principle

Greedy colouring: order vertices, assign smallest colour not used by neighbours.

Step-by-Step Explanation

  • Define proper colouring and chromatic number.
  • Greedy algorithm: pick vertex order.
  • Assign minimum available colour.
  • Repeat for all vertices.
  • Note: greedy may not give minimum colours.

Architecture / Diagram

Triangle graph K3 needs 3 colours (each connected to other two)

Algorithm

Greedy(G):
  for v in vertex_order:
    colour[v] = min colour not used by neighbours

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Exam timetable: each exam slot = colour; conflicting exams cannot share slot.

Advantages

  • Simple greedy approach
  • Upper bound on chromatic number
  • Many heuristics exist

Disadvantages

  • Optimal colouring is NP-hard
  • Greedy depends on vertex order

Applications

  • Timetabling
  • Register allocation
  • Map colouring
  • Sudoku

Exam Tips

Always define chromatic number. Give greedy example with 4-node graph.

Common Mistakes

  • Saying greedy always gives optimal (it does not).

Difference Table

Proper colouring=no adjacent same | Chromatic number=min colours needed

Time Complexity

Greedy O(V+E); optimal colouring NP-hard

Space Complexity

O(V) colour array

Interview Questions

  • Why is graph colouring NP-hard?

Memory Tricks

Graph Colouring = no neighbour same colour
