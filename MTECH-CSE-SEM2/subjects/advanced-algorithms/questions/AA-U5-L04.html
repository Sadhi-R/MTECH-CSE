Question

Explain randomized algorithms.

DefinitionRandomized algorithms use random choices during execution; output may vary but probability of correct answer is high.

Visual Example

Introduction

Las Vegas: always correct, random runtime. Monte Carlo: fixed time, small error probability.

Why It Is Needed

Often simpler and faster than deterministic for hard problems.

Working Principle

Flip coins to guide choices; repeat or amplify success probability.

Step-by-Step Explanation

  • Classify: Las Vegas vs Monte Carlo.
  • Example: Randomized QuickSort — random pivot, O(n log n) expected.
  • Example: Monte Carlo primality test.
  • Amplification: run k times, error ≤ (1/2)^k.
  • Compare with deterministic counterpart.

Architecture / Diagram

Randomized QuickSort: random pivot → O(n log n) expected vs O(n²) worst deterministic

Algorithm

RandomizedSelect: random pivot partition → expected O(n)

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Randomized min-cut (Karger's) finds minimum cut with high probability.

Advantages

  • Often simpler expected time
  • Good approximation ratios

Disadvantages

  • Error probability in Monte Carlo
  • Randomness source needed

Applications

  • QuickSort
  • Primality testing
  • Approximation algorithms

Exam Tips

Give Las Vegas vs Monte Carlo definitions with one example each.

Common Mistakes

  • Saying randomized always means wrong answer sometimes (Las Vegas is always correct).

Difference Table

Las Vegas=always correct | Monte Carlo=bounded error

Time Complexity

Space Complexity

O(1) random bits per step typically
