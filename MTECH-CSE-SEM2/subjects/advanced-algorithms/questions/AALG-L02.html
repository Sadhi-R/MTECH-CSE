Minimum Spanning Tree: Kruskal vs Prim

High WeightageQuestion Bank93%Unit-1Long
Question

Explain MST and compare Kruskal and Prim algorithms with complexity.

DefinitionMST connects all vertices with minimum total edge weight and no cycle.

Introduction

Two classic greedy methods solve MST efficiently.

Why It Is Needed

MST is used in low-cost network design.

Working Principle

Kruskal sorts edges; Prim grows one tree from a start node.

Step-by-Step Explanation

  • Define cut property
  • Explain Kruskal with DSU
  • Explain Prim with heap
  • Compare

Algorithm

Flowchart

Input graph -> choose min safe edge repeatedly -> n-1 edges.

Real-Time Example

Designing cheapest fiber network among campus blocks.

Advantages

  • Optimal by cut property
  • Practical and fast

Disadvantages

  • Need connected graph for full tree
  • Kruskal sorting overhead

Applications

  • Power grids
  • Road planning
  • Sensor networking

Exam Tips

Mention cut property and cycle property for full marks.

Common Mistakes

  • Writing Prim without priority queue complexity update.

Difference Table

Kruskal suits sparse edge list; Prim suits dense adjacency structures.

Time Complexity

Kruskal O(ElogE), Prim O(ElogV) with heap.

Space Complexity

O(V) for parent/key; DSU O(V).
