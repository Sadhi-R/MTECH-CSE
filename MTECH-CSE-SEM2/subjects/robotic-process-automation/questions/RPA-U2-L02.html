Question

Explain dashboard panel features.

DefinitionDashboard panel in Control Room provides real-time visibility: bot status, queue depth, device utilization, and SLA metrics.

Visual Example

Introduction

Widgets show running/completed/failed bots, trends, and alerts.

Why It Is Needed

Operations teams monitor automation health without digging into logs.

Working Principle

CR aggregates execution data from runners; dashboard renders KPIs and drill-down links.

Step-by-Step Explanation

  • Open Control Room dashboard.
  • View bot run status (success/fail/running).
  • Check device pool utilization.
  • Monitor queue backlog.
  • Drill into failed runs for errors.

Architecture / Diagram

Dashboard widgets: Status | Devices | Queues | SLA

Algorithm

Login → Dashboard → filter by date/bot → drill-down logs

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Ops manager sees 3 failed invoice bots; drills to 'Element not found' error.

Advantages

  • At-a-glance health
  • Proactive failure detection
  • SLA tracking

Disadvantages

  • Needs proper widget configuration
  • Stale data if runners offline

Applications

  • NOC-style monitoring
  • Automation CoE reporting
  • Executive KPI views

Exam Tips

List 4-5 dashboard widgets and what each shows.

Common Mistakes

  • Ignoring failed bot alerts on dashboard.

Difference Table

Dashboard=real-time ops | Audit log=historical detail

Time Complexity

Real-time refresh on execution events

Space Complexity

CR analytics database

Interview Questions

  • Dashboard vs Audit log?

Memory Tricks

Dashboard = bot status, devices, queues, SLA

5-Mark Answer

Widget list. Metrics. Drill-down. Ops use case.

10-Mark Answer

Control Room dashboard: bot status, device utilization, queues, SLA widgets.
