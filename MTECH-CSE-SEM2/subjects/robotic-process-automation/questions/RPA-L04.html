Devices and Bot Runners

Question Bank90%Unit-2Long
Question

Explain device pools, Bot Runner assignment, and unattended automation.

DefinitionA device is a machine where bots execute; Bot Runner is licensed execution capability.

Introduction

Device pools help allocate bots dynamically to available machines.

Why It Is Needed

Good device strategy increases utilization and reliability.

Working Principle

Control Room maps bot runs to specific or pooled runners.

Step-by-Step Explanation

  • Define devices
  • Explain pool assignment
  • Discuss failover practices

Architecture / Diagram

Device pool diagram with queued tasks and available runners.

Algorithm

Task assignment algorithm based on runner availability.

Flowchart

Queue task -> pick free runner -> execute -> update status.

Real-Time Example

Cab booking system assigning nearest available driver.

Advantages

  • Higher utilization
  • Reduced manual intervention

Disadvantages

  • License planning required
  • Dependency on machine readiness

Applications

  • Night batch processing
  • Back-office unattended tasks

Exam Tips

Mention device health checks and lock state requirements.

Common Mistakes

  • Assigning critical bots to single point of failure device.

Difference Table

Dedicated device gives predictability; pool gives flexibility.

Time Complexity

Runtime depends on queue depth and bot duration.

Space Complexity

Device metadata and execution logs.

  • Asked in implementation-focused interviews.
