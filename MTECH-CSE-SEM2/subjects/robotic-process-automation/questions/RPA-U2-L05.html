Question

Explain Control Room panel functions.

DefinitionControl Room panel functions include Bot Repository, Schedules, Work Queues, Devices, Users/Roles, Credentials, and Audit Log.

Visual Example

Introduction

Each panel serves a specific admin or developer function in the automation lifecycle.

Why It Is Needed

Knowing panels maps to operational tasks in exams and deployment.

Working Principle

Navigate panel → perform action (upload bot, create schedule, add credential) → effect on runners.

Step-by-Step Explanation

  • Bot Repository: store and version bots.
  • Schedules: time/event triggers.
  • Work Queues: transaction items for bots.
  • Devices: register runner machines.
  • Credentials: secure login storage.
  • Audit Log: all actions recorded.

Architecture / Diagram

Panels: Bots | Schedules | Queues | Devices | Credentials | Audit

Algorithm

Select panel → configure → save → runners pick up changes

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Create work queue 'Invoices', add items, bot processes each item from queue panel.

Advantages

  • Organized admin UI
  • Covers full lifecycle
  • Integrated credential mgmt

Disadvantages

  • Many panels — training needed
  • Misconfiguration affects all bots

Applications

  • Daily RPA operations
  • Admin certification exams
  • CoE standard procedures

Exam Tips

List each panel with one-line function — quick full marks.

Common Mistakes

  • Confusing Schedules (time) with Work Queues (transaction list).

Difference Table

Schedule=time trigger | Work Queue=data items

Time Complexity

Queue processing: FIFO per config

Space Complexity

CR DB per panel module

Interview Questions

  • Work queue vs schedule?

Memory Tricks

CR panels = bots, schedules, queues, devices, credentials, audit

5-Mark Answer

Each panel function. One example action per panel.

10-Mark Answer

Control Room panels: repository, schedules, queues, devices, credentials, audit log.

One-Line RevisionControl Room panels manage bots, schedules, work queues, devices, credentials, and audit.
