Question

Explain Web Control Room architecture.

DefinitionWeb Control Room is the browser-based central management console for Automation Anywhere bots, schedules, users, and credentials.

Visual Example

Introduction

Architecture: web UI → Control Room server → bot runners/devices → target applications.

Why It Is Needed

Central hub for enterprise RPA governance and orchestration.

Working Principle

Administrators deploy tasks, assign roles, schedule runs; bot agents pull work and report status.

Step-by-Step Explanation

  • User accesses Web CR via browser.
  • CR server stores bots, schedules, credentials.
  • Bot Runner/Control Room Agent on target machine.
  • CR assigns workload to available devices.
  • Audit logs and dashboards track execution.

Architecture / Diagram

Browser→Web CR Server→Bot Runners→Applications

Algorithm

Login CR → upload/deploy task → schedule → runner executes → logs to CR

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Admin schedules nightly report bot on 5 runner machines from Web CR dashboard.

Advantages

  • Centralized management
  • Role-based access
  • No desktop CR install needed for admin

Disadvantages

  • Requires network connectivity
  • Server single point of governance

Applications

  • Enterprise bot deployment
  • Multi-team automation
  • Compliance auditing

Exam Tips

Draw 3-tier: Web UI, CR Server, Bot Runners.

Common Mistakes

  • Confusing Web CR with Bot Agent (admin vs execution).

Difference Table

Web CR=management | Bot Runner=execution

Time Complexity

Schedule triggers per cron/queue

Space Complexity

CR DB + credential vault

Interview Questions

  • Control Room vs Bot Agent?

Memory Tricks

Web CR = central bot management hub

Architecture layers. Components. Deployment flow.

Web Control Room architecture: browser UI, server, bot runners, credential vault.
