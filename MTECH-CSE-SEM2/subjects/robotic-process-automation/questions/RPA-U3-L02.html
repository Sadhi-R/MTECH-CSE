Question

Explain relevance of audit log.

DefinitionAudit log in Control Room records all user actions, bot executions, configuration changes, and login events with timestamps.

Visual Example

Introduction

Immutable trail for compliance, troubleshooting, and security investigations.

Why It Is Needed

Regulated enterprises require audit logs for SOX, HIPAA, and internal governance.

Working Principle

Every CR action writes log entry: who, what, when, outcome; searchable and exportable.

Step-by-Step Explanation

  • Automatic logging of CR events.
  • Bot run start/end/status captured.
  • User login and permission changes logged.
  • Filter/search by date, user, bot.
  • Export for compliance reports.

Architecture / Diagram

Audit entry: [2025-01-15 10:00] User Admin deployed Bot_Invoice v3

Algorithm

Action occurs → CR writes audit record → admin searches/filters

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Auditor exports Q4 logs showing all credential access and bot deployments.

Advantages

  • Compliance ready
  • Forensic troubleshooting
  • Accountability

Disadvantages

  • Large logs need retention policy
  • Requires secure access control

Applications

  • SOX compliance
  • Security incident investigation
  • CoE governance reviews

Exam Tips

List what gets logged: user actions, bot runs, config changes.

Common Mistakes

  • Disabling audit for convenience (compliance violation).

Difference Table

Audit log=who did what when | Bot log=execution detail

Time Complexity

Search/filter O indexed query

Space Complexity

CR audit database

Interview Questions

  • Audit log vs bot run log?

Memory Tricks

Audit log = compliance trail of all CR actions

5-Mark Answer

What is logged. Search. Compliance use. Example entry.

10-Mark Answer

Audit log relevance: compliance, security, troubleshooting all CR and bot events.
