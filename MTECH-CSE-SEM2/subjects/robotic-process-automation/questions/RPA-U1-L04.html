Question

Explain benefits and limitations of RPA.

DefinitionRPA automates rule-based digital work using software bots.

Visual Example

Introduction

Enterprise adoption driven by cost and accuracy gains.

Why It Is Needed

Humans spend 60-80% time on repetitive tasks; bots work 24/7.

Working Principle

Bot mimics human UI interaction: login, read screen, enter data, submit.

Step-by-Step Explanation

  • List benefits with examples.
  • List limitations honestly.
  • Give bank/hospital/HR examples.
  • Conclude when RPA fits vs not.

Architecture / Diagram

Human task → Bot replicates → 24/7 execution → Audit log

Algorithm

Identify rule-based process → Record → Test → Deploy → Monitor

Flowchart

Rule-based? → Yes:RPA | No: consider AI/human

Real-Time Example

Hospital: bot extracts discharge codes, fills insurance claim form.

Advantages

  • 60-80% cost saving
  • Near-zero errors
  • 24/7
  • Audit trail
  • Fast ROI

Disadvantages

  • No emotional judgment
  • UI dependency
  • Exception handling
  • Maintenance

Applications

  • Banking
  • Insurance
  • HR
  • Payroll
  • Customer support

Exam Tips

Must give BOTH benefits AND limitations with industry examples.

Common Mistakes

  • Saying RPA replaces all human jobs or handles all decisions.

Difference Table

RPA=UI automation | Traditional integration=API level

Time Complexity

ROI often 3-12 months

Space Complexity

Cloud/on-prem infrastructure

Interview Questions

  • When NOT to use RPA?

Memory Tricks

Benefits ASSC (Accuracy Speed Scale Compliance)

5-Mark Answer

Define RPA + 4 benefits + 3 limitations + 2 examples.

10-Mark Answer

Full evaluation with bank/hospital cases, architecture, limits, ROI discussion.

One-Line RevisionRPA wins on speed/accuracy/24-7; loses on complex judgment and UI changes.
