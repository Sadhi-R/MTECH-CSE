Question

Explain FTP, PGP, and terminal commands.

DefinitionFTP Command transfers files to/from servers; PGP Command encrypts/decrypts files; Terminal Emulator automates mainframe/green-screen apps.

Visual Example

Introduction

Enterprise integration commands for secure file exchange and legacy systems.

Why It Is Needed

Many enterprises still use FTP drops, encrypted files, and terminal apps.

Working Principle

FTP: connect, get/put files. PGP: encrypt with public key before send. Terminal: send keys, read screen text.

Step-by-Step Explanation

  • FTP: Connect → Login → Get/Put File → Disconnect.
  • PGP: Encrypt File with recipient key / Decrypt with private key.
  • Terminal: Connect session → Send Text/Key → Extract screen → Disconnect.
  • Chain: download via FTP → decrypt PGP → process.
  • Log all transfers in audit.

Architecture / Diagram

FTP get → PGP decrypt → Excel process → PGP encrypt → FTP put

Algorithm

Secure pipeline: fetch encrypted → decrypt → automate → encrypt → upload

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

Bank statement: FTP download .pgp → decrypt → Terminal post entries → encrypt ack → FTP upload.

Advantages

  • FTP standard file exchange
  • PGP ensures confidentiality
  • Terminal covers legacy apps

Disadvantages

  • FTP plain text unless SFTP
  • PGP key management overhead
  • Terminal screen parsing fragile

Applications

  • Banking file exchange
  • Mainframe integration
  • B2B encrypted payloads

Exam Tips

One workflow chaining FTP + PGP + Terminal.

Common Mistakes

  • Using FTP without SFTP for sensitive data.

Difference Table

FTP=transfer | PGP=encrypt | Terminal=mainframe UI

Time Complexity

Transfer time depends on file size/network

Space Complexity

Key ring + temp decrypted files

Memory Tricks
