Question

Explain activity and bots management.

DefinitionActivity and bots management covers creating, versioning, deploying, scheduling, and monitoring bots and their activity logs in Control Room.

Visual Example

Introduction

Activities = audit trail of user and bot actions; Bots = packaged automation tasks.

Why It Is Needed

Lifecycle management ensures only approved bots run in production.

Working Principle

Developer uploads bot → admin approves → schedule/queue assignment → runner executes → activity logged.

Step-by-Step Explanation

  • Upload bot (.atmx) to repository.
  • Check-in/check-out for version control.
  • Create schedule or work queue.
  • Assign to device pool.
  • Review activity tab for run history.

Architecture / Diagram

Bot repo → Schedule/Queue → Device → Activity log

Algorithm

Develop → Upload → Deploy → Schedule → Monitor activity

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

HR bot v2.1 uploaded; schedule weekdays 6 AM; activity shows 45 successful runs.

Advantages

  • Version control in CR
  • Central scheduling
  • Full activity audit

Disadvantages

  • Version conflicts if unchecked
  • Orphan schedules if bot deleted

Applications

  • Production bot governance
  • Compliance audits
  • Team collaboration

Exam Tips

Describe bot lifecycle: upload → deploy → schedule → activity.

Common Mistakes

  • Running unapproved bot version in production.

Difference Table

Activity=log trail | Bot=automation package

Time Complexity

Schedule per cron or queue trigger

Space Complexity

Bot repository + activity DB

Interview Questions

  • Schedule vs work queue?

Memory Tricks

Activity + bots = upload, schedule, monitor runs
