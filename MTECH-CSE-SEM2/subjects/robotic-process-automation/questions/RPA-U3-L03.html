Question

Explain administration capabilities.

DefinitionAdministration capabilities in Control Room include user/role management, license management, server settings, and tenant configuration.

Visual Example

Introduction

Admins control who can create, run, or view bots and system-wide policies.

Why It Is Needed

Strong admin controls prevent unauthorized automation and credential exposure.

Working Principle

Admin assigns roles (AAE_Admin, Bot Creator, Bot Runner); configures SMTP, retention, integration.

Step-by-Step Explanation

  • Create/manage users and roles (RBAC).
  • Configure system settings (email, retention).
  • Manage licenses and feature entitlements.
  • Set password policies and session timeout.
  • Configure Git/integration for bot sync.

Architecture / Diagram

Admin → Users/Roles → Policies → System Settings

Algorithm

Admin login → Users panel → assign role → configure policies

Flowchart

Start -> Define -> Process -> Validate -> Conclude -> End

Real-Time Example

CoE admin creates 'Finance Bot Creator' role with upload-only to Finance folder.

Advantages

  • Fine-grained RBAC
  • Central policy enforcement
  • License visibility

Disadvantages

  • Over-permissioned users risk
  • Misconfiguration affects all users

Applications

  • Enterprise IT administration
  • CoE onboarding
  • Multi-team CR setup

Exam Tips

List roles: Admin, Creator, Runner, Viewer with permissions.

Common Mistakes

  • Giving all users Admin role.

Difference Table

Admin=govern users/settings | Developer=create bots

Time Complexity

Admin tasks infrequent; impact high

Space Complexity

User directory + policy config
